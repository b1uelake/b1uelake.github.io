# 今日学习内容

### 变量、列表、字典、循环、函数
变量 = 存数据的盒子
列表 = `[1, 2, 3]` 有顺序、可改
字典 = `{"name": "chen"}` 键值对
循环 = `for i in list:` / `for k, v in dict.items():`
函数 = `def my_func(x): return x*2`

### 文件读写、json 处理

f的意思是file object（文件对象）。

| 动作  | 代码                                | 比喻      |
| --- | --------------------------------- | ------- |
| 打开书 | `open("xxx.json", "r")`           | 把书拿起来   |
| 看书  | `f.read()` / `json.load(f)`       | 看书里的内容  |
| 写字  | `f.write()` / `json.dump(data,f)` | 把内容写进书里 |
| 合上书 | `f.close()` / 用 `with` 自动关闭       | 把书放回去   |

输入→输出

json.load（f）  = JSON文件 → Python数据
json.dump（data,f） = Python数据 → JSON文件

f.read() = 取出内容（文件 → 内存）  
f.write() = 存入内容（内存 → 文件）

### VS Code 断点 debug 基础

#### 什么是断点
断点 = 断点就是“暂停”，代码运行到断点行时暂停。

#### 打断点
在行号左边**点一下**，出现红点 🔴

#### 启动调试
在左侧工具栏，图案是开始键上有一个小虫子

#### 看变量
- **左边变量窗口**：显示当前所有变量
- **代码行末尾**：内联显示变量值
- **鼠标悬停**：停在变量上弹出值

#### Debug 找 bug 流程
1. 运行程序，发现结果不对
2. 在可疑位置打断点
3. 启动调试，程序停在断点处
4. 看变量值，找错误
5. 往回追踪调用处，找到 bug 位置
6. 修复
