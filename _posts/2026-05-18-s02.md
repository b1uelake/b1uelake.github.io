在单工具 agent 的基础上，今天搞清楚了如何用 dispatch map（分发字典）让 agent 支持多工具——加新工具不需要改循环，只需要注册一个 handler。

---

## 核心概念

**Dispatch Map（分发字典）**

- **直觉/类比**（必填）：像餐厅的叫号系统。厨房不认识每一桌客人，只根据号码牌（工具名）找到对应厨师（handler 函数）。新开一个菜品档口，只需挂上新号码牌，叫号的流程一字不改。
- **解决什么问题**：没有它，每加一个工具就要在循环里写一个 `if/elif`。工具一多，循环变成一堵墙，改一处容易动到别处。
- **和已有知识的关系**：是对 s01 硬编码 bash 调用的替代，本质是策略模式（Strategy Pattern）——把"选谁"和"怎么做"解耦。

**safe_path（路径沙箱）**

- **直觉/类比**（必填）：小区门禁。不管来者说什么理由，门卫只认工作区内的地址，`../../../etc/passwd` 这类越界路径一律拒绝，不给任何协商余地。
- **解决什么问题**：纯 bash 没有边界，模型理论上可以读写系统任意位置。`safe_path()` 把权限从"整台机器"收窄到"一个文件夹"。

---

## 架构图

```
+--------+      +-------+      +------------------+
|  User  | ---> |  LLM  | ---> | Tool Dispatch    |
| prompt |      |       |      | {                |
+--------+      +---+---+      |   bash: run_bash |
                    ^           |   read: run_read |
                    |           |   write: run_wr  |
                    +-----------+   edit: run_edit |
                    tool_result | }                |
                                +------------------+
```

这张图的核心关系：LLM 只负责"说出工具名和参数"，真正执行的是右边的 dispatch map。两者通过 `tool_result` 形成闭环，但 LLM 不感知任何 handler 的内部实现——它只知道工具名。

## 代码

### `safe_path` + `run_read`

**输入**：文件相对路径（字符串）+ 可选的行数上限。 **输出**：文件文本内容，最多 50000 字符。 **最关键的一行**：`if not path.is_relative_to(WORKDIR)` — 唯一的安全检查，所有越界请求在这里终止，没有任何降级处理。

```python
def safe_path(p: str) -> Path:
# `WORKDIR` `WORKDIR` 是 **Work Directory** 的缩写，是你启动 Agent 时设定的工作目录，比如你的项目文件夹
# `p` 是模型传入的文件路径，比如模型说要读 `"hello.py"`
    path = (WORKDIR / p).resolve()# 把路径转成**绝对路径**-->/project/../etc/passwd展开 `../`
    if not path.is_relative_to(WORKDIR):  # 关键：越界直接抛错，不协商
        raise ValueError(f"Path escapes workspace: {p}")
    return path

# 字符串 → (splitlines) → 列表 → ([:limit]截断) → (join) → 字符串
def run_read(path: str, limit: int = None) -> str:
    text = safe_path(path).read_text()
    lines = text.splitlines()# `splitlines()` 以换行符为分隔，把整段文字切成一个列表，每个元素是一行。
    if limit and limit < len(lines):
        lines = lines[:limit]
    return "\n".join(lines)[:50000]# 限制字符数为50000
```
代码解释：
```python

# 参数注解
def func(name: str, age: int):

# 变量注解
x: int = 5

# 返回值注解
def func() -> str:
```

```python

/home/user/project
         + ..  →  /home/user        ← "project" 消掉了

# 等价于
```

### Dispatch Map + Agent Loop

**输入**：模型响应中的工具名（`block.name`）和参数字典（`block.input`）。 **输出**：handler 的返回值字符串，或 `"Unknown tool: ..."` 错误信息。 **最关键的一行**：`TOOL_HANDLERS.get(block.name)` — 一次字典查找替代所有 `if/elif`，循环体本身对工具数量无感知。

```python
# **表示解包，把字典的每个 key-value 变成 `key=value` 的形式传给函数。
# dispatch map 不是定义函数，只是把**已有函数**和**工具名**对应起来。
# kw[]是必填，kw.get()是选填，若没有limit则返回None
TOOL_HANDLERS = {
    "bash":       lambda **kw: run_bash(kw["command"]),
    "read_file":  lambda **kw: run_read(kw["path"], kw.get("limit")),
    "write_file": lambda **kw: run_write(kw["path"], kw["content"]),
    "edit_file":  lambda **kw: run_edit(kw["path"], kw["old_text"], kw["new_text"]),
}

for block in response.content:
    if block.type == "tool_use":
        handler = TOOL_HANDLERS.get(block.name)  # 关键：找不到返回 None，不崩。拿名字去查对应的lambda函数。
        output = handler(**block.input) if handler \
            else f"Unknown tool: {block.name}"
        results.append({
            "type": "tool_result",
            "tool_use_id": block.id,
            "content": output,
        })
```


**运行时最重要的现象**：换用 `read_file` 之后，读大文件的输出长度变得可预期——`run_read` 有明确的截断逻辑，而 `cat` 没有。
### 代码解释：
`**` 有两个作用，取决于用在哪里：

| 位置                     | 作用           |
| ---------------------- | ------------ |
| 调用函数时 `test(**d)`      | 拆开字典，变成关键字参数 |
| 定义函数时 `def test(**kw)` | 收集关键字参数，变成字典 |

```python
# 普通函数
def my_func(**kw):
    return run_read(kw["path"], kw.get("limit"))

# 等价的 lambda，**定义一个匿名函数，接收任意关键字参数存入 kw 字典**。返回定义函数的运行结果。
lambda **kw: run_read(kw["path"], kw.get("limit"))
```

```python
# 定义时
lambda **kw: ...
# kw 还是空的，只是告诉 Python "我会用 **kw 接收参数"

# 调用时
handler(command="ls -la")
# 这时 kw 才真正有值：{"command": "ls -la"}
```


解包完整流程：

**第一步：** 循环拿到 LLM 的工具调用
```python
block.name = "read_file"
block.input = {"path": "hello.py", "limit": 100}
```

**第二步：** 查 dispatch map，找到对应 lambda
```python
handler = TOOL_HANDLERS["read_file"]
# handler = lambda **kw: run_read(kw["path"], kw.get("limit"))
```

**第三步：** 调用 handler，`**` 解包 block.input
```python
handler(**block.input)
# 等价于
handler(path="hello.py", limit=100)
```

**第四步：** lambda 用 `**kw` 接住，kw 变成字典
```python
kw = {"path": "hello.py", "limit": 100}
```

**第五步：** 从 kw 取参数，调用 run_read
```python
run_read(kw["path"], kw.get("limit"))
# 等价于
run_read("hello.py", 100)
```

解包发生了两次：

- `**block.input` — 字典解包成关键字参数
- `**kw` — 关键字参数收回成字典

## 术语表

|术语|含义|
|---|---|
|`TOOL_HANDLERS`|dispatch map，工具名 → handler 函数的字典|
|`safe_path()`|路径沙箱函数，拦截所有逃出工作区的路径|
|`WORKDIR`|工作区根目录，`safe_path` 的边界基准|
|`block.name`|模型响应中工具调用块的工具名字段|
|`block.input`|模型传给工具的参数字典|
|handler|单个工具的处理函数，dispatch map 的值|
