# s04 · Subagent 上下文隔离

今天在 s03 单一 Agent 循环的基础上，搞清楚了为什么要让子 Agent 用独立的 `messages[]`，以及这个"用完即扔"的设计在做什么事。

---

## 真实困惑

父 Agent 收到的是纯文本摘要，但如果子 Agent 的工具调用中途失败了、或者摘要不够准确，父 Agent 完全看不到过程细节——这个信息损失怎么权衡？目前还没想清楚边界在哪里。

---

## 核心概念

### Subagent 上下文隔离（Context Isolation）

- **直觉/类比**（必填）：像派一个临时工去仓库盘货。他在里面翻箱倒柜、记了满满一本草稿，最后只递出来一张纸条："共 312 件"。你拿到结论，他的草稿直接扔掉。主办公室桌面始终干净。
    
- **解决什么问题**：Agent 工作越久，`messages[]` 里堆的内容越多——读文件、跑命令的输出全永久留着。问"用什么测试框架"可能要读 5 个文件，但父 Agent 其实只需要一个词："pytest"。没有隔离，这 5 个文件的内容会永远占据主对话的上下文。
    
- **和已有知识的关系**：是对 s03 单循环的**扩展**，不是替代。父 Agent 还在跑自己的 `while tool_use` 循环，只是多了一个 `task` 工具可以外包子任务。
    

---

### task 工具（任务派发工具）

- **直觉/类比**：父 Agent 的工具箱里多了一个"外包按钮"。按下去，一个新员工从零开始干活，干完只汇报结果，过程不留痕迹。
    
- **解决什么问题**：让父 Agent 的 `messages[]` 只存高层摘要，不被中间过程污染。
    

---

## 结构图

```
Parent agent                     Subagent
+------------------+             +------------------+
| messages=[...]   |             | messages=[]      | <-- fresh
|                  |  dispatch   |                  |
| tool: task       | ----------> | while tool_use:  |
|   prompt="..."   |             |   call tools     |
|                  |  summary    |   append results |
|   result = "..." | <---------- | return last text |
+------------------+             +------------------+
Parent context stays clean. Subagent context is discarded.
```

**这张图的核心关系**：左边父 Agent 的 `messages[]` 只进一条摘要；右边子 Agent 的整个消息历史（无论多长）在返回后直接丢弃——信息是单向、有损压缩的。

---

## 关键代码

 父 Agent 有一个 `task` 工具。Subagent 拥有除 `task` 外的所有基础工具 (禁止递归生成)。

```python
PARENT_TOOLS = CHILD_TOOLS + [
    {"name": "task",
     "description": "Spawn a subagent with fresh context.",
     "input_schema": {
         "type": "object",
         "properties": {"prompt": {"type": "string"}},
         "required": ["prompt"],
     }},
]
```

| 字段           | 作用               |
| ------------ | ---------------- |
| `properties` | 定义"有哪些字段，各是什么类型" |
| `required`   | 定义"哪些字段必须填"      |

两个必须配合使用——`properties` 是"菜单"，`required` 是"必点菜"。

### `run_subagent()`

**输入**：一段自然语言 `prompt`（字符串） **输出**：子 Agent 最后一轮回复中的纯文本（字符串摘要）

**最关键的一行**：`sub_messages = [{"role": "user", "content": prompt}]`——子 Agent 从空列表启动，而不是继承父 Agent 的历史，这一行决定了隔离的本质。

**运行结果里最重要的现象**：子 Agent 可能跑了 30 次工具调用，但父 Agent 的 `messages[]` 里只多了一条 `tool_result`，长度几乎不变。

```python
def run_subagent(prompt: str) -> str:
    # 关键：空列表，子 Agent 不继承任何历史
    sub_messages = [{"role": "user", "content": prompt}]
    
    for _ in range(30):  # 安全上限，防止无限循环
        response = client.messages.create(
            model=MODEL, system=SUBAGENT_SYSTEM,
            messages=sub_messages,
            tools=CHILD_TOOLS,  # 注意：没有 task 工具，禁止递归
            max_tokens=8000,
        )
        sub_messages.append({"role": "assistant", "content": response.content})
        
        if response.stop_reason != "tool_use":  # stop_reason（停止原因）不是工具调用就退出
            break
        
        results = []
        for block in response.content:
            if block.type == "tool_use":
                handler = TOOL_HANDLERS.get(block.name)
                output = handler(**block.input)
                results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": str(output)[:50000]  # 截断防止单条结果过大
                })
        sub_messages.append({"role": "user", "content": results})
    
    # sub_messages 整个丢弃，只返回最后一段文字
    return "".join(
        b.text for b in response.content if hasattr(b, "text")#hasattr是has attribute，一个**过滤条件**，意思是：只取 `response.content` 里**有 `.text` 属性**的 block
    ) or "(no summary)"
```

## 工具权限对比

|组件|工具集|
|---|---|
|父 Agent|`CHILD_TOOLS` + `task`（可外包）|
|子 Agent|`CHILD_TOOLS` 全部，**但没有** `task`|

子 Agent 没有 `task` 工具，是刻意设计的——防止子 Agent 再生子 Agent，避免递归失控。

---

## 术语表

|术语|含义|
|---|---|
|`Harness 层`|包裹模型调用的脚手架代码，负责循环、工具分发、上下文管理|
|`PARENT_TOOLS`|父 Agent 的工具列表，包含子 Agent 没有的 `task`|
|`CHILD_TOOLS`|子 Agent 的工具列表，基础文件/命令工具，无递归权限|
|`stop_reason`|停止原因，API 返回字段；值为 `"tool_use"` 说明模型还想调工具，否则本轮对话结束|
|`sub_messages`|子 Agent 的私有消息历史，函数返回后随栈帧销毁|

## s03 → s04 变更一览

|维度|s03|s04|
|---|---|---|
|工具数量|5 个（共享）|5个基础（子端）+ `task`（父端专属）|
|上下文|单一共享 `messages[]`|父子各自独立|
|子任务机制|无|`run_subagent()`|
|父 Agent 收到的|工具原始输出|仅摘要文本|
