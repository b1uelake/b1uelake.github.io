# 更深的网络——Linear层与BatchNorm层


## 模块1：今日概览

### 日期与主题
训练一个更深的网络——从零手写 `Linear`（线性层）和 `BatchNorm1d`（批归一化层），模拟 PyTorch 中 `nn.Module` 的 API 风格。

### 与上一篇的衔接
昨天我们已经构建了基本的网络前向传播流程，理解了单层网络如何计算输出。今天我们往"更深"迈进：手动实现两个核心层类，让它们像 PyTorch 原生模块一样可以被组合、调用和管理参数。

### 核心问题
1. **Linear 层的权重初始化为什么要除以 $\sqrt{f_{an\_in}}$？不除会怎样？**
2. **BatchNorm 到底在做什么？训练和推理时行为为什么不同？running_mean / running_var 是怎么来的？**

---

## 模块2：概念精讲


### 概念1：Linear（线性层 / 全连接层）

- **一句话定义**：对输入做一次线性变换 $y = xW + b$，就是"乘一个权重矩阵，加上一个偏置向量"。

- **为什么要用它**：每一层 Linear 是网络的"基本计算单元"。没有它，网络根本无法学习任何从输入到输出的映射关系。堆叠多层 Linear（中间加非线性激活）就是深度学习最基础的结构。

- **与已有知识的关系**：这是对之前手动矩阵乘法 `x @ W` 的封装。之前我们直接操作张量，现在把它包装成一个"类"，让网络结构更清晰、更模块化。这是从"脚本式代码"到"面向对象代码"的一步进化。

- **类比/直觉**：Linear 层就像一个"筛子工厂"——输入数据是原料，权重矩阵是筛网的形状，偏置是出厂校准值。不同的权重矩阵会筛出不同的特征。

#### 关键细节：权重初始化中的 `/ fan_in**0.5`

```python
self.weight = torch.randn((fan_in, fan_out)) / fan_in**0.5
```

这行代码做的是 **Kaiming 初始化的简化版**。直觉解释：

- 假设输入 $x$ 有 $fan\_in$ 个元素，每个元素独立且方差为 1。
- 那么 $x \cdot W$ 的每一个输出元素是 $fan\_in$ 个随机变量的加权求和，方差会膨胀为 $fan\_in$ 级别。
- 除以 $\sqrt{fan\_in}$ 可以把输出的方差重新拉回到约 1。
- **如果不除**：随着网络层数加深，每层输出的方差会指数级增长（爆炸）或缩小（消失），梯度同样如此——这就是**梯度爆炸/消失问题**。

---

### 概念2：BatchNorm1d（批归一化）

- **一句话定义**：在一个 mini-batch 内，对每个特征维度做归一化（减均值、除标准差），然后再用可学习的缩放参数 $\gamma$ 和偏移参数 $\beta$ 做仿射变换。

- **为什么要用它**：
  - 解决**内部协变量偏移（Internal Covariate Shift）**问题——随着训练进行，每一层输入的分布不断变化，导致后面的层很难学。
  - 让每一层的输入保持在相对稳定的分布上，**加速训练收敛**。
  - 有轻微的**正则化效果**（因为 mini-batch 的均值和方差有噪声）。

- **与已有知识的关系**：和我们之前做的输入特征归一化（将训练集均值/方差应用到所有数据）是同一个思想。区别在于 BatchNorm 是**在每一层内部**、**对每个 mini-batch** 动态做归一化，而不是只在输入层做一次。

- **类比/直觉**：想象你在一家工厂的流水线上，每道工序都会改变零件的尺寸。BatchNorm 就是每道工序后面加一个"标准化卡尺"——不管上一道工序怎么变，我先把零件校准到统一尺寸，再交给下一道工序。

#### 训练 vs 推理：两种模式

| | 训练时 | 推理时 |
|---|---|---|
| 均值 / 方差 | 用**当前 batch** 的统计量 | 用**训练期间累积的 running_mean / running_var** |
| 原因 | 需要适应最新数据分布 | 推理时可能只有单个样本，无法算 batch 统计量 |

**running_mean 和 running_var** 是通过**指数移动平均（EMA）**更新的：

$$
\text{running\_mean} = (1 - m) \times \text{running\_mean} + m \times \text{batch\_mean}
$$

其中 $m$ 就是 `momentum`（注意：这里的 momentum 和优化器的 momentum 不是同一个东西！）。$m = 0.1$ 意味着新 batch 的均值只占 10% 的权重，旧的 running_mean 占 90%——这使得 running 统计量是一个**平滑的、全局的趋势**。

---

### 概念3：`__call__` 与 `parameters` 方法的 API 设计

- **`__call__`**：让对象可以像函数一样被调用（`layer(x)` 而不是 `layer.forward(x)`）。这是 PyTorch `nn.Module` 的惯例——`nn.Module.__call__` 内部会调用 `forward()`，同时自动处理 hook 等逻辑。
- **`parameters`**：返回该层所有需要训练的参数列表。好处是上层代码可以用一个统一的接口收集所有参数，传给优化器。**不需要手动检查这一层有没有 bias**——`parameters()` 方法自己处理了。

## 模块3：代码实现

### 代码段1：Linear 层实现

**功能说明**：实现一个全连接层，接受输入张量 $x$（形状 `[batch_size, fan_in]`），输出张量形状 `[batch_size, fan_out]`。

```python
class Linear:
    def __init__(self, fan_in, fan_out, bias=True):
        # 权重初始化：标准正态 / sqrt(fan_in)，保持输出方差稳定
        self.weight = torch.randn((fan_in, fan_out), generator=generator) / fan_in**0.5
        # 偏置初始化为零（初期不引入额外偏移）
        self.bias = torch.zeros(fan_out) if bias else None

    def __call__(self, x):
        self.out = x @ self.weight          # 矩阵乘法：[N, fan_in] @ [fan_in, fan_out] = [N, fan_out]
        if self.bias is not None:
            self.out += self.bias            # 广播加偏置
        return self.out

    def parameters(self):
        # 返回所有可训练参数；用三元表达式处理有无 bias 的情况
        return [self.weight] + ([] if self.bias is None else [self.bias])
```

**后置总结**：
- `fan_in**0.5` 初始化确保每层输出方差 ≈ 1，这是训练深网络能收敛的前提条件之一。
- `parameters()` 方法返回的是**引用**（不是拷贝），所以优化器更新这些张量时，类内部的 `self.weight` 也会同步更新。

### 代码段2：BatchNorm1d 层实现

**功能说明**：对输入张量 $x$（形状 `[batch_size, dim]`）做批归一化，输出形状不变。

```python
class BatchNorm1d:
    def __init__(self, dim, eps=1e-5, momentum=0.1):
        self.dim = dim
        self.eps = eps                          # 防止除以零的小常数
        self.momentum = momentum                # EMA 的更新速率
        self.training = True                    # 标志位：训练模式 or 推理模式

        # === 可训练参数（通过反向传播更新）===
        self.gamma = torch.ones(dim)            # 缩放因子，初始化为 1（相当于一开始不做缩放）
        self.beta = torch.zeros(dim)            # 偏移因子，初始化为 0（相当于一开始不做偏移）

        # === 缓冲变量（通过 EMA 更新，不参与反向传播）===
        self.running_mean = torch.zeros(dim)    # 全局均值的估计
        self.running_var = torch.ones(dim)      # 全局方差的估计（初始化为 1）

    def __call__(self, x):
        if self.training:
            # 训练模式：用当前 batch 的统计量
            xmean = x.mean(0, keepdim=True)     # 沿 batch 维度求均值，shape: [1, dim]
            xvar = x.var(0, keepdim=True)        # 沿 batch 维度求方差，shape: [1, dim]
        else:
            # 推理模式：用训练期间累积的 running 统计量
            xmean = self.running_mean
            xvar = self.running_var

        # 标准化：(x - 均值) / 标准差
        xhat = (x - xmean) / torch.sqrt(xvar + self.eps)

        # 仿射变换：gamma * xhat + beta（可学习的缩放和偏移）
        self.out = self.gamma * xhat + self.beta

        # 更新 running 统计量（只在训练模式下更新）
        with torch.no_grad():  # 注意：应加 no_grad，这些不需要梯度
            self.running_mean = (1 - self.momentum) * self.running_mean + self.momentum * xmean
            self.running_var = (1 - self.momentum) * self.running_var + self.momentum * xvar

        return self.out

    def parameters(self):
        return [self.gamma, self.beta]
```

**后置总结**：
- BatchNorm 只有 2 个可训练参数：$\gamma$（缩放）和 $\beta$（偏移），`running_mean` 和 `running_var` 不是通过梯度更新的。
- 输出的 `self.out` 会被后续层继续使用，所以放在 `self.out` 上方便调试时查看中间值。

### 踩坑记录（代码中的 Bug）

课程原始代码中存在以下问题，我在整理时已修正：

| # | 原始代码 | 问题 | 修正 |
|---|---|---|---|
| 1 | `x @ welf.weight` | 拼写错误，`welf` 应为 `self` | `x @ self.weight` |
| 2 | `self.out += self.bias` 在 `__call__` 中 | 没有问题，但注意 bias 的 shape 是 `[fan_out]`，通过广播机制自动扩展到 `[batch_size, fan_out]` | — |
| 3 | `momentum` 直接使用 | 应为 `self.momentum`，否则会引用外部变量或报错 | 改为 `self.momentum` |
| 4 | running 统计量更新未包裹 `torch.no_grad()` | EMA 更新不需要计算梯度，不加 `no_grad()` 会浪费计算资源并影响计算图 | 加上 `with torch.no_grad()` |

> **经验**：从课程/教程复制代码时，一定要逐行审查，不要盲目运行。拼写错误（`welf` → `self`）是最隐蔽的 bug。

## 模块4：数学公式与代码的对应

### 对应关系 1：Linear 层

$$
y = xW + b
$$

| 数学符号 | 代码变量 | 含义 |
|---|---|---|
| $x$ | `x` | 输入张量，shape `[N, fan_in]` |
| $W$ | `self.weight` | 权重矩阵，shape `[fan_in, fan_out]` |
| $b$ | `self.bias` | 偏置向量，shape `[fan_out]` |
| $y$ | `self.out` | 输出张量，shape `[N, fan_out]` |

**直觉**：这个公式在做"坐标空间的旋转和拉伸"——把输入从 $fan\_in$ 维空间映射到 $fan\_out$ 维空间。

---

### 对应关系 2：BatchNorm 的标准化

$$
\hat{x} = \frac{x - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}
$$

$$
y = \gamma \hat{x} + \beta
$$

| 数学符号 | 代码变量 | 含义 |
|---|---|---|
| $\mu_B$ | `xmean` | 当前 batch 的均值 |
| $\sigma_B^2$ | `xvar` | 当前 batch 的方差 |
| $\epsilon$ | `self.eps` | 数值稳定性常数（防除零） |
| $\gamma$ | `self.gamma` | 可学习的缩放参数（初始为 1） |
| $\beta$ | `self.beta` | 可学习的偏移参数（初始为 0） |
| $\hat{x}$ | `xhat` | 归一化后的中间结果 |

**直觉**：先"标准化"（把数据拉到均值 0、方差 1 的分布），再"反标准化"（用 $\gamma$ 和 $\beta$ 恢复网络需要的分布）。关键是：**标准化的均值和方差是网络自己学到的，不是人工固定的**。

---

### 对应关系 3：Running Mean 的 EMA 更新

$$
\text{running\_mean}_{\text{new}} = (1 - m) \cdot \text{running\_mean}_{\text{old}} + m \cdot \text{batch\_mean}
$$

| 数学符号 | 代码变量 | 含义 |
|---|---|---|
| $m$ | `self.momentum` | 更新速率（本例为 0.1） |
| $\text{batch\_mean}$ | `xmean` | 当前 batch 的均值 |

**直觉**：这就像一个"记忆系统"——90% 记住历史趋势，10% 吸收新信息。momentum 越小，running 统计量越稳定但反应越慢；越大则越敏感但越不稳定。

## 模块5：今日总结

### 收获
1. **权重初始化不是"玄学"**：除以 $\sqrt{fan\_in}$ 是为了保持前向传播中每层输出的方差稳定，这是训练深网络的工程基础。
2. **BatchNorm 的核心是"训练时用 batch 统计，推理时用全局统计"**：通过 EMA 在训练过程中平滑地累积全局统计量，确保推理时行为稳定。
3. **面向对象封装的价值**：`__call__` + `parameters()` 的 API 设计让层可以像乐高积木一样组合，上层代码不需要关心每层的内部实现细节。


### 术语表更新

| 术语 | 含义 |
|---|---|
| `fan_in` | 层输入的特征维度数 |
| `fan_out` | 层输出的特征维度数 |
| BatchNorm（批归一化） | 在 mini-batch 内对每个特征做归一化，加速训练收敛 |
| `gamma` ($\gamma$) | BatchNorm 的可学习缩放参数 |
| `beta` ($\beta$) | BatchNorm 的可学习偏移参数 |
| `running_mean` | 训练过程中通过 EMA 累积的全局均值估计，推理时使用 |
| `running_var` | 训练过程中通过 EMA 累积的全局方差估计，推理时使用 |
| EMA（指数移动平均） | 一种平滑更新机制，新值只占小部分权重，旧值占大部分 |
| `momentum`（此处） | EMA 中新值的权重，注意与优化器 SGD 的 momentum 不是同一个概念 |
| `eps` ($\epsilon$) | 防止除以零的微小常数 |
| `__call__` | Python 魔术方法，让对象可以像函数一样被调用 |
