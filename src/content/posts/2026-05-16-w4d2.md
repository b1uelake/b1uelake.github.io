# 网络架构搭建与权重初始化策略

**日期与主题**：搭建完整的多层神经网络架构，学习 Tanh 激活函数的实现、批量归一化（BatchNorm）的使用，以及一个关键的训练技巧——通过调整最后一层权重来改善模型校准（Calibration）。

**与上一篇的衔接**：昨天我们已经实现了 Linear 层和 BatchNorm1d 层的核心代码（包括前向传播和参数管理）。今天我们把这些积木拼装成一个完整的深层网络，并学习一个重要的初始化技巧。

**核心问题**：

## 模块2：概念精讲

### Tanh 激活函数

- **一句话定义**：Tanh（双曲正切）是一个把任意实数压缩到 $(-1, 1)$ 区间内的非线性函数，公式为 $\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$。

- **为什么要用它**：如果没有激活函数，多层线性层的堆叠等价于一个线性变换（矩阵乘法可以合并），网络就失去了学习非线性关系的能力。Tanh 在这里负责引入非线性。在 ReLU 出现之前，Tanh 是深度学习中最常用的激活函数之一。

- **与已有知识的关系**：之前在手动构建的 MLP 中可能用过 ReLU。Tanh 是另一种选择，它的输出以 0 为中心（mean ≈ 0），这一点比 Sigmoid（输出在 0-1 之间）更有利于后续层的学习。这个网络选择 Tanh，是为了和 BatchNorm 配合——BatchNorm 把数据归一化到接近 0 均值，Tanh 在 0 附近接近线性（梯度接近 1），两者配合很好。

- **类比/直觉**：想象一个"智能音量旋钮"，不管输入的音量多大或多小（正无穷或负无穷），输出都会被限制在 -1 到 1 之间。输入越大越接近 1，输入越小越接近 -1，中间区域（接近 0）则几乎是线性放大的。

**Tanh 的梯度特性**（重要补充）：

$$\frac{d(\tanh(x))}{dx} = 1 - \tanh^2(x)$$

当 $\tanh(x)$ 接近 ±1 时，梯度趋近于 0（梯度饱和/消失）。这也是为什么网络不能太深（除非用 BatchNorm 来缓解），也是后来 ReLU 兴起的原因之一。

---
### 模型校准（Calibration）

- **一句话定义**：模型校准是指模型输出的置信度（概率）与实际正确率是否一致。一个校准良好的模型，当它说"我 80% 确定是 A"时，它的正确率就真的应该是 80%。

- **为什么要用它**：神经网络（尤其是随机初始化的）在训练初期往往"过度自信"——Softmax 输出的概率分布过于尖锐（某个类的概率接近 1）。这会导致交叉熵损失在初期非常大，训练不稳定。

- **与已有知识的关系**：这和 Loss 在训练初期非常大是同一个问题的不同角度。减小最后一层的 gamma（BatchNorm 的缩放参数），相当于把最后一层的输出幅度缩小，使 Softmax 的输入更接近均匀分布，从而降低初始置信度。

- **类比/直觉**：想象一个刚入职的医生，对每种疾病的诊断都很自信（"100% 是感冒！"），但实际上他对各种疾病并没有判断力。让他变得"谦虚"一些（"可能 10% 是感冒，10% 是过敏..."），反而更接近真实情况，后续学习也更有效率。

---

### 权重缩放因子 5/3（为什么用 `5/3` 而不是 `1.0`）

- **一句话定义**：这是 Kaiming 初始化的一种手动近似——对于 Tanh 激活函数后的方差保持，理想的缩放因子约为 $\sqrt{\frac{1}{1 - (2/\pi)}} \approx 1.0$（Tanh 的方差），但在实际实践中常用 `5/3` 来稍微放大信号，补偿 Tanh 对方差的"挤压"效应。

- **为什么要用它**：在网络深层传播时，如果不做特殊处理，每经过一层 Tanh，信号的方差都会被压缩。`5/3` 的缩放因子能补偿这种压缩，使深层网络中各层的激活值保持合理的方差范围，避免信号逐层衰减（消失）。

- **与已有知识的关系**：这和之前学的"为什么 Linear 层的初始化很重要"是同一类问题。好的初始化 = 保持信号在传播过程中方差稳定。

- **类比/直觉**：就像接力赛跑中，每一棒都会稍微减速。如果知道每棒大概会慢多少，就可以让前面的选手跑得稍快一些来补偿。`5/3` 就是那个"补偿系数"。

> **注意**：代码中当前使用的是 `1.0`（注释掉了 `5/3`），这意味着当前版本没有做方差补偿，配合 BatchNorm 使用时问题不大，因为 BatchNorm 会重新归一化方差。

## 模块3：代码实现

### 代码段1：Tanh 激活函数类

**前置说明**：实现一个简单的 Tanh 包装类，接口与 Linear、BatchNorm1d 统一（有 `__call__` 和 `parameters`），这样可以和其他层无缝串联。

```python
class Tanh:
    def __call__(self, x):
        self.out = torch.tanh(x)  # 记录输出，方便后续调试/可视化
        return self.out

    def parameters(self):
        return []  # Tanh 没有可学习参数，但必须实现这个方法
        # 否则后面遍历 layers 调用 parameters() 时会报 AttributeError
```

**后置总结**：
- Tanh 层没有参数（不需要学习），所以 `parameters()` 返回空列表。
- `self.out` 的保存是为了方便后续分析——比如查看激活值的分布（是否饱和）。
- **设计模式理解**：这是一个典型的"鸭子类型"（Duck Typing）设计——所有层都遵循相同的接口（`__call__` 做前向传播，`parameters()` 返回参数），这样上层代码可以用统一的方式遍历任何层。

**踩坑记录**：如果不实现 `parameters()` 方法，后面 `[p for layer in layers for p in layer.parameters()]` 这行代码会在遇到 Tanh 层时报 `AttributeError: 'Tanh' object has no attribute 'parameters'`。这是接口一致性的重要体现。

---

### 代码段2：网络架构组装

**前置说明**：将之前实现的 Linear、BatchNorm1d 和今天的 Tanh 组装成一个 6 层（含输出层）的深层网络。

```python
n_embd = 10       # 每个 token 的嵌入维度
n_hidden = 100    # 隐藏层神经元数量
g = torch.Generator().manual_seed(2147483647)  # 固定随机种子，保证可复现

# 嵌入矩阵：将词表中的每个字符映射为 n_embd 维向量
C = torch.randn((vocab_size, n_embd), generator=g)

# 网络结构：5 个隐藏层 + 1 个输出层
layers = [
    # 隐藏层1：输入维度 = n_embd * block_size（上下文中所有token的嵌入拼接）
    Linear(n_embd * block_size, n_hidden, bias=False), BatchNorm1d(n_hidden), Tanh(),
    # 隐藏层2-5：维度不变，都是 100 -> 100
    Linear(n_hidden, n_hidden, bias=False), BatchNorm1d(n_hidden), Tanh(),
    Linear(n_hidden, n_hidden, bias=False), BatchNorm1d(n_hidden), Tanh(),
    Linear(n_hidden, n_hidden, bias=False), BatchNorm1d(n_hidden), Tanh(),
    Linear(n_hidden, n_hidden, bias=False), BatchNorm1d(n_hidden), Tanh(),
    # 输出层：从隐藏维度映射到词表大小（每个字符一个分数）
    Linear(n_hidden, vocab_size, bias=False), BatchNorm1d(vocab_size)
]
```

**后置总结**：

每层的结构是 `Linear → BatchNorm → Tanh`（输出层没有 Tanh），这个组合有明确的设计逻辑：
1. **Linear** 做线性变换
2. **BatchNorm** 归一化，稳定训练
3. **Tanh** 引入非线性

输出层只有 `Linear → BatchNorm`，**没有 Tanh**，因为我们希望输出是 logits（任意实数），最终通过 Softmax 转为概率。

**关于 `bias=False`**：当使用了 BatchNorm 时，Linear 层的 bias 是多余的。因为 BatchNorm 有自己的偏移参数 $\beta$（代码中的 `beta`），它会重新减去均值再加回偏移，Linear 的 bias 会被 BatchNorm 的减均值操作抵消掉。所以设为 `False` 可以减少参数量且不影响表达能力。

---

### 代码段3：权重初始化调整（关键技巧）

**前置说明**：对网络的参数做特殊初始化，目的是改善模型训练初期的校准（Calibration）。

```python
with torch.no_grad():  # 禁用梯度追踪，因为这是初始化，不是训练
    # 技巧1：缩小最后一层的 gamma（BatchNorm的缩放因子）
    # 效果：使输出层的 logits 更小 → Softmax 输出更接近均匀分布 → 初始置信度更低
    layers[-1].gamma *= 0.1

    # 技巧2：对前面的隐藏层 Linear 做权重缩放
    for layer in layers[:-1]:
        if isinstance(layer, Linear):
            layer.weight *= 1.0  # 当前是 1.0（即不缩放）
            # 注释中的 5/3 是推荐值，用于补偿 Tanh 的方差压缩
```

**后置总结**：

这里有两个独立的技巧，解决两个不同的问题：

| 技巧 | 目标层 | 作用 | 解决的问题 |
|------|--------|------|-----------|
| `gamma *= 0.1` | 最后一层（输出层） | 缩小输出 logits 的幅度 | 训练初期 Loss 爆炸/过度自信 |
| `weight *= 5/3` | 隐藏层 Linear | 补偿 Tanh 的方差压缩 | 深层网络信号衰减 |

**为什么用 `torch.no_grad()`**：初始化操作不应该被计算图追踪，否则 PyTorch 会记录不必要的计算历史，浪费内存。

**踩坑记录**：
- `layers[-1].gamma` 是 BatchNorm1d 的缩放参数。如果网络最后一层不是 BatchNorm，这个方法就不适用。代码中最后一层确实是 BatchNorm1d(vocab_size)，所以可以直接操作。
- `isinstance(layer, Linear)` 这个检查是必要的——因为 layers 列表里混合了 Linear、BatchNorm1d 和 Tanh 三种对象，我们只想修改 Linear 的权重。

---

### 代码段4：参数收集与梯度设置

**前置说明**：将嵌入矩阵 C 和所有层的参数收集到一个列表中，并统一开启梯度追踪。

```python
# 合并所有可学习参数：嵌入矩阵 + 各层参数
parameters = [C] + [p for layer in layers for p in layer.parameters()]

# 查看总参数量
print(sum(p.nelement() for p in parameters))
# nelement() 返回张量中的元素总数（不是维度数，是所有维度的乘积）

# 为所有参数开启梯度追踪
for p in parameters:
    p.requires_grad = True
```

**后置总结**：
- `nelement()` 和 `numel()` 功能相同，都是返回元素总数。
- 参数总数 = 嵌入矩阵参数 + 所有 Linear 的 weight + 所有 BatchNorm 的 gamma 和 beta。
- **注意**：Tanh 层不会贡献任何参数（`parameters()` 返回空列表），所以在列表推导式中会被自动跳过。
- 和之前只有一层的小网络相比，这个深层网络的参数量会大得多——这就是"容量"的直观体现。

## 模块4：数学公式与代码的对应

### 公式1：Tanh 前向传播

$$\text{Tanh}(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$$

对应代码：`self.out = torch.tanh(x)`

对应变量：$x$ = 上一层（BatchNorm）的输出

直觉解释：把任意数值挤压到 -1 到 1 之间，输入越大越接近 1，越小越接近 -1。

---

### 公式2：最后一层 gamma 缩放

$$\gamma_{\text{new}} = 0.1 \times \gamma_{\text{old}}$$

对应代码：`layers[-1].gamma *= 0.1`

对应变量：$\gamma$ = BatchNorm 的缩放参数

直觉解释：把输出层的"音量"调小 10 倍，让 Softmax 的输入不那么极端，初始预测更"谦虚"。

---

### 公式3：权重缩放（可选的 5/3 初始化）

$$W_{\text{new}} = \frac{5}{3} \times W_{\text{old}}$$

对应代码：`layer.weight *= 1.0`（注释中建议用 `5/3`）

对应变量：$W$ = Linear 层的权重矩阵

直觉解释：在 Tanh 压缩信号之前先把信号放大一点，这样经过 Tanh 后方差不会缩小太多，保持信号在深层网络中的"强度"。

## 模块5：今日总结

### 收获

1. **Tanh 的实现极其简洁**——本质上就是 `torch.tanh(x)` 包一个壳，但这个壳必须和 Linear、BatchNorm 保持相同的接口（`__call__` + `parameters`），这是面向接口编程的体现。

2. **输出层 gamma 缩小 0.1 是解决训练初期 Loss 爆炸的实用技巧**——本质上是降低初始置信度，让模型从"什么都不知道"的状态出发，而不是"胡乱自信"的状态。

3. **`bias=False` 在 BatchNorm 网络中是有道理的**——因为 BatchNorm 会重新计算均值并减去，Linear 的 bias 会被浪费掉。
