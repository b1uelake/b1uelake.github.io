# 反向传播训练循环、批量归一化的推理模式与Python装饰器


## 模块1：今日概览

**日期与主题**：神经网络训练循环的完整流程（backward pass → 参数更新 → 统计追踪），以及训练结束后如何切换到推理模式使用批量归一化（Batch Normalization）；附带学习Python装饰器（decorator）语法。

**与上一篇的衔接**：前几天我们完成了网络的前向传播（forward pass）、损失计算和批量归一化的训练实现。今天我们补上训练循环的"后半段"——反向传播怎么更新参数、学习率怎么变化、训练结束后如何固化BN的统计量用于评估。

**核心问题**：
1. 训练循环中反向传播、梯度清零、参数更新这三步到底在干什么？学习率为什么要中途衰减？

## 模块2：概念精讲

### 2.1 反向传播与梯度清零（Gradient Zeroing）

**一句话定义**：每次更新参数之前，先把所有参数的 `.grad` 属性清零，再调用 `loss.backward()` 重新计算梯度。

**为什么要用它**：PyTorch的 `.backward()` 默认是**累加**梯度到 `.grad` 上，而不是覆盖。如果不清零，多次调用 `.backward()` 后梯度会越叠越大，参数更新就会发散。你可以理解为：旧梯度是"上次的记忆"，如果不擦掉，新旧梯度会混在一起，导致"方向混乱"。

**与已有知识的关系**：这是对Day 2手写反向传播的PyTorch实现。手写版我们自己逐步计算 `dlogits`、`dh`、`dw2`……PyTorch版只需一行 `loss.backward()`，它自动通过计算图（computational graph）完成同样的事情。本质是同一件事。

**类比/直觉**：就像考试涂答题卡——每答一题之前，必须先把上一题的痕迹擦干净，否则两题答案重叠，机器读不出来。

---

### 2.2 学习率衰减（Learning Rate Decay）

**一句话定义**：训练过程中人为地逐步降低学习率。这里用的是"阶梯式"策略——前100000步用 `lr=0.1`，之后切换到 `lr=0.001`。

**为什么要用它**：训练前期参数离最优解很远，需要大步快走（大学习率）快速接近；后期参数已经在"最优解附近"了，大步会来回震荡甚至跳出最优区域，需要小步微调。类比开车：高速公路上油门踩满，快到目的地时必须减速。

**与已有知识的关系**：这是优化策略（optimization strategy）的一部分。我们之前学了SGD的更新公式 `p.data += -lr * p.grad`，学习率 `lr` 是其中唯一的"手动旋钮"。

**代码对应**：

```python
lr = 0.1 if i < 100000 else 0.001
```

这里 `i` 是当前训练步数。注意：这种硬切换（hard switch）比较粗糙，更常见的是指数衰减、余弦退火等平滑策略。但核心思想一样——后期降速。

### 2.3 批量归一化的推理模式（BN Inference Mode）

**一句话定义**：训练结束后，用整个训练集算出的**全局均值和全局标准差**来替代推理/评估时的batch统计量，使BN的行为确定化。

**为什么要用它**：这是BN最微妙的地方。训练时，每个batch有自己的均值和方差（因为batch是随机采样的，每个batch的统计量不同）。但推理时：
- 你可能一次只输入一个样本（batch size = 1），此时均值就是样本本身，方差为0——BN完全失效。
- 即使batch size > 1，推理时的batch组成也是不确定的，导致输出不稳定。

所以我们需要**固定的**、**确定性的**均值和方差。做法是：训练过程中用**指数移动平均（running mean / running std）**持续追踪全局统计量，训练结束后"冻结"这些值。

**与已有知识的关系**：这正是Day笔记中BN前向传播的"推理版本"。训练时：
```python
hpreact = bngain * (hpreact - hpreact.mean(0, keepdim=True)) / hpreact.std(0, keepdim=True) + bnbias
```
推理时把 `hpreact.mean()` / `hpreact.std()` 替换为 `bnmean_running` / `bnstd_running`。

**类比/直觉**：训练时BN像"课堂小测验"——每次考试的平均分都不同（因为题目/学生不同）。推理时BN像"用全校历届平均分做标准"——这个标准是固定的、有代表性的。

**代码中的关键操作**：

```python
with torch.no_grad():
    # 用整个训练集计算一次，得到"全局统计量"
    emb = C[Xtr]
    embcat = emb.view(emb.shape[0], -1)
    hpreact = embcat @ w1 + b1
    bnmean = hpreact.mean(0, keepdim=True)
    bnstd = hpreact.std(0, keepdim=True)
```

> **注意**：这里 `bnmean` / `bnstd` 是手动算的。PyTorch的 `nn.BatchNorm1d` 内部会自动维护 `running_mean` / `running_var`，通过 `momentum` 参数做指数移动平均。这段代码是教学用的手动实现。

`with torch.no_grad()` 的作用：告诉PyTorch不要追踪这段计算的梯度（节省内存，也避免意外修改参数）。

### 2.4 Python装饰器（Decorator）

**一句话定义**：装饰器是一种"函数包装器"——`@A` 放在函数 `B` 定义前面，等价于 `B = A(B)`，即用函数 `A` 包装 `B`，返回一个增强版的 `B`。

**为什么要用它**：避免重复写样板代码。比如"关闭梯度计算"这个需求，如果每个评估函数都要手动写 `with torch.no_grad()`，很繁琐。装饰器让你在函数定义时一行搞定。

**两种语法的区别**：

```python
# 语法1：@A（不带括号）—— A本身是装饰器函数
@A
def B(): ...
# 等价于 B = A(B)

# 语法2：@A()（带括号）—— A() 的返回值是装饰器函数
@A()
def B(): ...
# 等价于 tmp = A(); B = tmp(B)
```

关键区别：带括号时，`A()` 先被**调用**，返回值（一个函数）才是真正的装饰器。不带括号时，`A` 本身就是装饰器。

**与已有知识的关系**：这和深度学习无直接关系，但代码中 `@torch.no_grad()` 就是装饰器的应用——它把 `split_loss` 函数包装成"在无梯度模式下运行"的版本。

**类比/直觉**：装饰器像"给函数穿外套"。`@torch.no_grad()` 就是给函数穿上一件"不要算梯度"的外套。函数本身不变，但行为被修改了。

**完整示例解析**：

```python
def double_call(func):
    def wrapper():
        func()  # 第一次调用
        func()  # 第二次调用
    return wrapper

@double_call
def hello():
    print('hi')

hello()
# 输出：
# hi
# hi
```

执行流程：
1. `@double_call` 修饰 `hello` → `hello = double_call(hello)`
2. `double_call(hello)` 内部：`func` 参数接收了原始的 `hello`，返回 `wrapper`
3. 所以 `hello` 现在指向 `wrapper`
4. 调用 `hello()` 实际上调用 `wrapper()` → 打印两次 `hi`

### 2.5 `split_loss` 评估函数详解

**一句话定义**：一个用装饰器包装的评估函数，接受 `'train'`/`'val'`/`'test'` 字符串参数，计算并打印对应数据集上的损失。

**为什么要用它**：训练过程中我们只看训练集loss，但真正关心的是模型在**未见过的数据**上的表现（泛化能力）。这个函数让我们方便地检查三个集的loss。

**代码逐行解析**：

```python
@torch.no_grad()  # 关闭梯度追踪，评估不需要反向传播
def split_loss(split):
    # 用字典映射选择数据集
    x, y = {
        'train': (Xtr, Ytr),
        'val':   (Xdev, Ydev),
        'test':  (Xte, Yte),
    }[split]
    
    # 前向传播（和训练时完全一样）
    emb = C[x]                                    # 查嵌入表
    embcat = emb.view(emb.shape[0], -1)           # 展平
    hpreact = embcat @ w1 + b1                    # 线性变换
    
    # 关键区别：使用running统计量，而非当前batch统计量
    hpreact = bngain * (hpreact - bnmean_running) / bnstd_running + bnbias
    
    h = torch.tanh(hpreact)                       # 激活
    logits = h @ w2 + b2                          # 输出层
    loss = F.cross_entropy(logits, y)             # 损失
    print(split, loss.item())                     # 打印结果
```

**踩坑记录**：
- 如果用 `bnmean`（当前batch的统计量）而非 `bnmean_running`，评估结果会因为batch不同而波动，不可复现。
- `split_loss` 中注释掉的两行（`bnmean = ...` / `bnstd = ...`）就是"训练时的做法"，留作对比。推理时必须用 `bnmean_running` / `bnstd_running`。

## 模块3：代码实现

### 3.1 训练循环核心三步

**前置说明**：这是每个训练步（step）的核心操作。输入是当前batch的数据，输出是更新后的参数。每一步包含三个子操作：清零梯度→计算梯度→更新参数。

```python
# 第一步：梯度清零
for p in parameters:
    p.grad = None        # PyTorch中None比0更高效（释放内存）

# 第二步：反向传播，计算所有参数的梯度
loss.backward()          # 自动从loss出发，沿计算图反向计算每个参数的梯度

# 第三步：参数更新（SGD）
lr = 0.1 if i < 100000 else 0.001   # 学习率衰减
for p in parameters:
    p.data += -lr * p.grad           # 沿梯度反方向移动一步
```

**后置总结**：这三步构成一个完整的训练迭代。`p.data` 直接修改参数值（绕过autograd的追踪）。注意是 `p.data += ...` 而不是 `p += ...`，后者会重新构建计算图节点。

**变量名追踪**：
- `parameters`：所有需要更新的参数列表（`C, w1, b1, bngain, bnbias, w2, b2`）
- `p`：循环中当前参数的临时名
- `i`：全局训练步计数器
- `lr`：学习率（learning rate），同一概念在不同语境下可能叫 `step_size` 或 `eta`

---

### 3.2 训练统计追踪

```python
# 每10000步打印一次当前loss
if i % 10000 == 0:
    print(f'{i:7d}/{max_step:7d}: {loss.item():.4f}')

# 记录log10(loss)用于后续画图
lossi.append(loss.log10().item())
```

**后置总结**：用 `log10` 而不是原始loss画图，是因为loss从几十几百快速降到个位数，原始尺度下前期变化被压缩到看不见。取对数后，不同量级的变化都能清晰展示。

**类比**：就像地图用比例尺，log变换就是对loss做一次"比例尺缩放"，让下降曲线更直观。

## 模块4：数学公式与代码的对应

### 4.1 SGD更新规则

$$
\theta_{t+1} = \theta_t - \eta \cdot \nabla_\theta \mathcal{L}
$$

| 符号 | 含义 | 代码对应 |
|------|------|----------|
| $\theta_t$ | 当前参数值 | `p.data` |
| $\eta$ | 学习率 | `lr` |
| $\nabla_\theta \mathcal{L}$ | 损失对参数的梯度 | `p.grad` |
| $\theta_{t+1}$ | 更新后的参数值 | 更新后的 `p.data` |

**直觉解释**：这个公式在做一件事——**沿着让损失减小最快的方向，走一小步**。梯度告诉你"哪个方向损失增大"，减号告诉你"走反方向"，学习率告诉你"走多远"。

### 4.2 BN推理时的归一化

$$
\hat{x} = \gamma \cdot \frac{x - \mu_{\text{running}}}{\sigma_{\text{running}}} + \beta
$$

| 符号                        | 含义        | 代码对应             |
| ------------------------- | --------- | ---------------- |
| $x$                       | 归一化前的线性输出 | `hpreact`        |
| $\mu_{\text{running}}$    | 训练集全局均值   | `bnmean_running` |
| $\sigma_{\text{running}}$ | 训练集全局标准差  | `bnstd_running`  |
| $\gamma$                  | 可学习的缩放参数  | `bngain`         |
| $\beta$                   | 可学习的偏移参数  | `bnbias`         |

**直觉解释**：用固定的"标准尺"把数据拉到同一尺度，然后给模型两个可调旋钮（gain和bias）让它自己决定合适的尺度和偏移。这和Day中训练时的BN是**同一个公式**，唯一区别是均值/方差的来源。

## 模块5：今日总结

### 收获

1. **梯度清零是必须的**：PyTorch默认累加梯度，不清零会导致梯度爆炸。`p.grad = None` 比 `p.grad.zero_()` 更推荐。
2. **BN推理模式的关键切换**：训练时用batch统计量（每个batch不同），推理时用全局running统计量（固定不变）。这是BN最容易踩坑的地方。
3. **装饰器本质是函数包装**：`@A` 就是 `B = A(B)`，没有什么魔法。

### 困惑

- `bnmean_running` / `bnstd_running` 这里是训练后一次性用全部训练集算的。但PyTorch官方的 `nn.BatchNorm1d` 是训练过程中每一步都做指数移动平均更新。两种方式的结果差异有多大？什么时候必须用EMA而不能用一次性计算？（猜测：如果训练数据量特别大、或者训练过程中数据分布有变化，EMA可能更稳定？）
- `p.data += -lr * p.grad` 为什么必须用 `.data`？如果直接写 `p = p - lr * p.grad` 会怎样？（初步理解：`=` 会创建新tensor，断开与计算图的关联；而 `.data` 是原地修改。但对 `p` 是列表元素的情况下赋值是否也断开？需要实验验证。）

### 术语表更新

| 术语 | 含义 |
|------|------|
| **backward pass** | 反向传播，从loss出发沿计算图反向计算每个参数的梯度 |
| **梯度清零（gradient zeroing）** | 将 `p.grad` 置零/None，防止梯度累加 |
| **学习率衰减（learning rate decay）** | 训练过程中逐步降低学习率 |
| **BN推理模式（inference mode）** | 使用固定的全局统计量（running mean/std）而非batch统计量 |
| **装饰器（decorator）** | Python语法糖，`@A` 等价于 `B = A(B)` |
| **`@torch.no_grad()`** | 装饰器，关闭梯度追踪，用于评估/推理阶段节省内存 |
| **`F.cross_entropy`** | PyTorch内置的交叉熵损失函数，内部包含softmax |
| **EMA（Exponential Moving Average）** | 指数移动平均，一种平滑统计量的方法，新数据权重更大 |
