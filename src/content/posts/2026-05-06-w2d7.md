我们已经完成了简单的MLP构建，但是现在有一些问题需要解决。

```python
import torch
import torch.nn.functional as F
import matplotlib.pyplot as plt # for making figures
%matplotlib inline

# read in all the words
words = open('names.txt', 'r').read().splitlines()

# build the vocabulary of characters and mappings to/from integers
chars = sorted(list(set(''.join(words))))
stoi = {s:i+1 for i,s in enumerate(chars)}
stoi['.'] = 0
itos = {i:s for s,i in stoi.items()}
vocab_size = len(itos)

# build the dataset
block_size = 3 # context length: how many characters do we take to predict the next one?

def build_dataset(words):
	X, Y = [], []
	for w in words:
		context = [0] * block_size
		for ch in w + '.':
		ix = stoi[ch]
		X.append(context)
		Y.append(ix)
		context = context[1:] + [ix] # crop and append
	
	X = torch.tensor(X)
	Y = torch.tensor(Y)
	
	return X, Y

import random
random.seed(42)

random.shuffle(words)
n1 = int(0.8*len(words))
n2 = int(0.9*len(words))

Xtr, Ytr = build_dataset(words[:n1]) # 80%
Xdev, Ydev = build_dataset(words[n1:n2]) # 10%
Xte, Yte = build_dataset(words[n2:]) # 10%


```


```python
# MLP revisited

n_embd = 10 # 字符嵌入向量的维度
n_hidden = 200 # 多层感知器（MLP）隐藏层中的神经元数量。

g = torch.Generator().manual_seed(2147483647) # 加入固定种子，为了可重复性
C = torch.randn((vocab_size, n_embd), generator=g)
W1 = torch.randn((n_embd * block_size, n_hidden), generator=g)
b1 = torch.randn(n_hidden, generator=g)
W2 = torch.randn((n_hidden, vocab_size), generator=g)
b2 = torch.randn(vocab_size, generator=g)

parameters = [C, W1, b1, W2, b2]
print(sum(p.nelement() for p in parameters)) # number of parameters in total
for p in parameters:
p.requires_grad = True
```


```python
# 和上次相同的优化设置

max_steps = 200000
batch_size = 32
lossi = []

for i in range(max_steps):
# 构建小批量数据
	ix = torch.randint(0, Xtr.shape[0], (batch_size,), generator=g)
	Xb, Yb = Xtr[ix], Ytr[ix] # 批量的输入X和标签Y
	
	# 前向传播
	emb = C[Xb] # 将字符嵌入为向量
	embcat = emb.view(emb.shape[0], -1) # 拼接向量
	hpreact = embcat @ W1 + b1 # 隐藏层激活前计算
	h = torch.tanh(hpreact) # 隐藏层
	logits = h @ W2 + b2 # 输出层
	
	loss = F.cross_entropy(logits, Yb) # 损失函数
	
	# 反向传播
	for p in parameters:
		p.grad = None # 清空梯度
	loss.backward() # 计算新梯度
	
	# 参数更新
	lr = 0.1 if i < 100000 else 0.01 # 阶梯式学习率衰减
	for p in parameters:
		p.data += -lr * p.grad # 现在 p.grad 一定不是 None
	
	# 记录统计数据
	if i % 10000 == 0: # 每隔一段时间打印日志
		print(f'{i:7d}/{max_steps:7d}: {loss.item():.4f}')
	lossi.append(loss.log10().item())
```

# 从模型中采样生成名字
g = torch.Generator().manual_seed(2147483647 + 10)  # 固定随机种子

for _ in range(20):  # 生成20个名字
    
    out = []
    context = [0] * block_size  # 初始化上下文为全0（起点标记）
    while True:
        # 前向传播

        out.append(ix)

            break
            
    # 解码并打印生成的名字

#### 问题1-初始loss过大

第一次训练时的loss很大，高达27，这会导致之后的训练模型出现问题。那么正常的loss应该是多少呢？第一次输入时，模型输出的概率分布是均匀的，即每个字符的概率都是 1/27，那么此时的
loss = - torch.tensor(1/27.0).log()应该是tensor(3.2958)才对，而不应该是27这么大。
问题出在哪里呢？
```python
# 4维示例，演示这个问题
logits = torch.tensor([0.0, 0.0, 0.0, 0.0])
probs = torch.softmax(logits, dim=0)
loss = -probs[2].log()
probs, loss# (tensor([0.2500, 0.2500, 0.2500, 0.2500]), tensor(1.3863))

logits = torch.tensor([-3.0, 5.0, 0.0, 2.0])
probs = torch.softmax(logits, dim=0)
loss = -probs[2].log()
probs, loss# (tensor([3.1741e-04, 9.4620e-01, 6.3754e-03, 4.7108e-02]), tensor(5.0553))

# 为什么这些数字的概率相差这么大，因为当数据不同时，经过softmax函数的指数运算之后，会极大的拉开原始数据之间的差别。只有数据都是同一个数时，概率才会相等，也就是说，由于初始的W2和b2是随机的，所以可能会导致输出的数据相差很大，于是最后输出的loss会很大。所以我们解决这个问题的方法就是让初始的最后一层MLP输出的值都是0--因为=0最简单-->W2和b2尽可能都是0
```
softmax函数
$$\text{Softmax}(z_i) = \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}$$
将神经网络的原始输出（Logits，数值向量）映射为概率分布（值在 0~1 之间，和为 1）。它常作为深度学习模型的最后一层。将最后一层的输出求指数，把所有数都变成正数，然后在除以他们的总和，得到概率。

dim=？-->代表着让？这个维度的和为0。
- **一维数据** (Vector)：只有 `dim=0`。
- **二维数据** (Matrix)：有 `dim=0`（行）和 `dim=1`（列）。
- **三维数据** (3D Tensor)：有 `dim=0`, `dim=1`, `dim=2`。

让我们回到micograd的代码中，看看我们是怎么使用tanh的返回值的

```python
def tanh(self):
    x = self.data
    t = (math.exp(2*x) - 1)/(math.exp(2*x) + 1)
    out = Value(t, (self, ), 'tanh')
    
    def _backward():
      self.grad += (1 - t**2) * out.grad
    out._backward = _backward
    
    return out
```
压缩后的tanh值为t，当t为1或-1时，1 - `t**2`=0，此时返回的梯度就消失了

横轴是200个神经元，纵轴代表32个样本，如果出现一整列32个样本都是白色的情况，说明这个神经元就会变成死神经元，因为所有的样本对应的梯度都会归0。

如何解决？--tanh过多的出现1/-1-->hpreact出现过多很大的数-->	hpreact = embcat @ W1 + b1 -->让W1，b1的值趋向于0
