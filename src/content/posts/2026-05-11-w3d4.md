# Day 6：makemore Part 5 — 从MLP到CNN（构建WaveNet风格的语言模型）

---

## 模块1：今日概览

**日期与主题**：用卷积神经网络（CNN）替代之前MLP的"拍扁"策略，构建层次化的字符级语言模型。

**核心问题**：

1. 为什么把上下文字符一次性"拍扁"是低效的？层次化组合为什么更好？
2. 卷积（Convolution）到底在做什么？为什么"reshape + Linear"就等于Conv1d？

---

## 模块2：概念精讲

### 增大上下文窗口 (block\_size: 3 → 8)

- **一句话定义**：模型能"看到"的前面字符数从3个增加到8个。
- **为什么要用它**：更大的上下文意味着更多信息。比如要预测"christo\_"后面的字符，只看前3个字符"chr"可能猜不出来，但看前8个"christo\_"就能有更强的线索。
- **与已有知识的关系**：这是 Day 2（makemore part 2）中 $\text{block\_size}$ 概念的直接扩展。
- **类比/直觉**：完形填空题，挖掉的词前面给3个字和给8个字，猜对的概率显然不同。

**带来的问题**：$\text{block\_size}=3$ 时，MLP 第一层输入维度是 $3 \times n\_\text{embd} = 30$；$\text{block\_size}=8$ 时变成 $8 \times n\_\text{embd} = 80$。参数量大幅增长。有没有更聪明的组合方式？

---

### 层次化特征组合 (Hierarchical Feature Composition)

- **一句话定义**：不把所有上下文一次性拍扁，而是先两两组合，再把组合结果两两组合，像一棵树一样逐层汇聚。
- **为什么要用它**：减少参数量（不是全连接，而是只让相邻元素交互），同时让模型自然学习局部模式（如"th"、"ch"这样的字母组合）。
- **与已有知识的关系**：这是对 MLP"全连接一切"思路的改进。MLP 中每个输入特征都连接到每个隐藏神经元，CNN 只让相邻特征交互，通过堆叠层数来扩大信息传播范围。
- **类比/直觉**：8个人传话。MLP 方式 = 把8个人拉到一个大会议室互相说话；层次化方式 = 先两人一组讨论（第1层），每组派代表再两两讨论（第2层），最后汇总（第3层）。第二种方式通信路径更短，效率更高。

字符序列:    h    e    l    l    o    .    .    .
            ───────────────────────────────────
第1层组合:  (h,e)   (l,l)   (o,.)   (.,.)     → 4个向量
第2层组合:  ((h,e),(l,l))   ((o,.),(.,.))     → 2个向量
第3层组合:  (((h,e),(l,l)),((o,.),(.,.)))      → 1个综合向量

> 这张结构图说明：3层两两组合就能把8个字符的信息汇聚到1个向量中。关键观察：每向上走一层，"看到"的原始输入数量翻倍（$2 \rightarrow 4 \rightarrow 8$），这就是感受野的指数增长。

---

### 1D卷积 (1D Convolution / Conv1d)

- **一句话定义**：用一个固定大小的滑动窗口在序列上滑动，在每个位置用**同一套权重**做线性变换，提取局部特征。
- **为什么要用它**：**参数共享**（同一窗口权重在所有位置复用）+ **局部连接**（只看相邻元素），比全连接层参数少得多。
- **与已有知识的关系**：Conv1d 本质上就是"分组 reshape + 线性变换"的高效封装。理解了 reshape 成 pair 再过线性层，就理解了卷积的核心。
- **类比/直觉**：一个固定大小的放大镜在句子上滑动，每到一个位置就观察"这一小片段"有什么特征。同一个放大镜在不同位置反复使用。

**数学本质**：对于输入序列 $x[0], x[1], \ldots, x[n]$，$\text{kernel\_size}=2$ 的卷积在位置 $i$ 的输出是：

$$
y[i] = W_0 \cdot x[i] + W_1 \cdot x[i+1] + b
$$

$W_0, W_1, b$ 在所有位置共享——这就是"参数共享"。

---
### 步幅卷积 (Strided Convolution)

- **一句话定义**：滑动窗口每次跳过 $\text{stride}$ 个位置再计算，输出序列变短。
- **为什么要用它**：快速压缩序列长度。$\text{block\_size}=8, \text{stride}=2$，3层后输出长度 $= 8 \div 2 \div 2 \div 2 = 1$，刚好汇聚成单个向量。
- **与已有知识的关系**：标准卷积（$\text{stride}=1$）的扩展。
- **类比/直觉**：你边走边看路标，$\text{stride}=1$ 是每走一步看一次，$\text{stride}=2$ 是每走两步看一次——你看的次数少了，但走得更快。

---

### 膨胀卷积 (Dilated Convolution) — 补充概念

- **一句话定义**：滑动窗口内部的元素之间插入"间隔"（dilation），窗口不变宽但能"看"到更远的位置。
- **为什么要用它**：在不减小空间分辨率的前提下增大感受野。WaveNet 的核心技术。
- **与已有知识的关系**：和 strided convolution 是增大感受野的两种不同策略。Stride 让序列变短，Dilation 保持序列长度不变。
- **类比/直觉**：Stride 像你迈大步走（位置跳着看）；Dilation 像你站住不动但把望远镜倍数调高（看得更远但中间有盲区）。


---

### 感受野 (Receptive Field)

- **一句话定义**：一个输出位置能"看到"多少个原始输入位置。
- **为什么要用它**：衡量模型的有效上下文范围。如果感受野不足，模型就无法利用远距离信息。
- **与已有知识的关系**：MLP 的感受野就是整个 $\text{block\_size}$（全连接天然覆盖），CNN 的感受野取决于层数和 $\text{kernel\_size}$，需要堆叠才能覆盖全部输入。
- **类比/直觉**：站在山谷里只能看到一小片天空（小感受野），爬到山顶就能看到整片天（大感受野）。每多一层卷积就像爬高一级。

**感受野增长**（$\text{kernel\_size}=2, \text{stride}=2$）：

| 层 | 每个输出看的上一层位置 | 覆盖的原始输入 |
|---|---|---|
| 第1层 | 2 | 2 |
| 第2层 | $2 \times 2 = 4$ | 4 |
| 第3层 | $2 \times 2 \times 2 = 8$ | **8（全覆盖）** |

→ 3层就够覆盖 $\text{block\_size}=8$ 的全部上下文！这是指数增长。

一般地，$L$ 层卷积的覆盖范围为：

$$
\text{感受野} = (\text{kernel\_size})^{L} = 2^{L}
$$

---

### CNN中的批量归一化 (BatchNorm1d)

- **一句话定义**：和之前 MLP 中的 BatchNorm 原理完全一样，只是输入变成3D张量 $\text{(batch, channels, length)}$，归一化在 $\text{(batch, length)}$ 维度上进行，每个 channel 有独立的均值/方差统计量。
- **为什么要用它**：稳定训练过程、允许更大学习率、有轻微正则化效果。
- **类比/直觉**：多门考试成绩标准化。每门课（channel）单独计算平均分和标准差，然后把所有学生的成绩标准化。

BatchNorm 在每个 channel $c$ 上做的事情：

$$
\hat{x}_c = \frac{x_c - \mu_c}{\sqrt{\sigma_c^2 + \epsilon}}
$$

$$
y_c = \gamma_c \hat{x}_c + \beta_c
$$

其中 $\mu_c, \sigma_c$ 是该 channel 在当前 batch 上的均值和标准差，$\gamma_c, \beta_c$ 是可学习的缩放和偏移参数。

---

### WaveNet 架构

- **一句话定义**：Google DeepMind 2016年提出的音频生成模型，用膨胀因果卷积实现层次化的序列建模。
- **为什么要用它**：用 $O(\log n)$ 深度的网络覆盖 $O(n)$ 的上下文窗口，比单层 MLP 更高效。
- **与已有知识的关系**：本节课所有概念的综合——层次化组合 + 1D卷积 + 逐层汇聚。
- **类比/直觉**：淘汰赛锦标赛——第一轮两两比赛，第二轮胜者再两两比赛，几轮后决出冠军。$\log_2(n)$ 轮就能处理 $n$ 个选手。

## 模块3：代码实现

### 代码段1：数据准备 — 增大 block\_size

**前置说明**：把上下文长度从3增加到8。输入变为8个字符索引，输出仍是下一个字符索引。和 Day 2 的数据准备代码结构相同，只改了 block\_size。

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import random

# 读取数据
words = open('names.txt', 'r').read().splitlines()

# 构建词表
chars = sorted(list(set(''.join(words))))
stoi = {s: i+1 for i, s in enumerate(chars)}
stoi['.'] = 0
itos = {i: s for s, i in stoi.items()}
vocab_size = len(itos)

block_size = 8  # 关键变化：从3增加到8

def build_dataset(words):
    X, Y = [], []
    for w in words:
        context = [0] * block_size  # 初始用0（'.'的索引）填充
        for ch in w + '.':
            ix = stoi[ch]
            X.append(context)        # 输入：前8个字符的索引
            Y.append(ix)             # 标签：下一个字符
            context = context[1:] + [ix]  # 滑动窗口：去头加尾
    X = torch.tensor(X)
    Y = torch.tensor(Y)
    return X, Y

# 划分数据集（和之前相同的 80/10/10 比例）
random.seed(42)
random.shuffle(words)
n1 = int(0.8 * len(words))
n2 = int(0.9 * len(words))

Xtr, Ytr = build_dataset(words[:n1])
Xdev, Ydev = build_dataset(words[n1:n2])
Xte, Yte = build_dataset(words[n2:])
```

**后置总结**：Xtr 的 shape 变为 $\text{(N, 8)}$，每个样本是8个字符索引。相比之前的 $\text{(N, 3)}$，每个预测能看到更多上下文。

---

### 代码段2：手动实现的层次化"卷积"（本讲核心教学代码）

**前置说明**：这是理解卷积本质的关键。不用 PyTorch 的 Conv1d，而是用 reshape + Linear 手动实现等价操作。输入是8个嵌入向量，输出是预测 logits。

```python
n_embd = 10     # 嵌入维度
n_hidden = 200  # 隐藏层维度

# 嵌入矩阵
C = torch.randn((vocab_size, n_embd))

# === 各层权重 ===
# 第1层：拼接相邻2个嵌入 (2*10=20) → 映射到200
W1 = torch.randn((2 * n_embd, n_hidden)) / (2 * n_embd)**0.5
b1 = torch.zeros(n_hidden)

# 第2层：拼接相邻2个隐藏向量 (2*200=400) → 映射到200
W2 = torch.randn((2 * n_hidden, n_hidden)) / (2 * n_hidden)**0.5
b2 = torch.zeros(n_hidden)

# 第3层：同上
W3 = torch.randn((2 * n_hidden, n_hidden)) / (2 * n_hidden)**0.5
b3 = torch.zeros(n_hidden)

# 输出层
W4 = torch.randn((n_hidden, vocab_size)) / n_hidden**0.5
b4 = torch.zeros(vocab_size)

parameters = [C, W1, b1, W2, b2, W3, b3, W4, b4]
for p in parameters:
    p.requires_grad = True

# === 前向传播 ===
emb = C[Xtr]  # (batch, 8, n_embd) — 8个字符各自查表得到嵌入

# --- 第1层：两两组合 ---
# 核心操作：view 把 (batch, 8, 10) 变成 (batch, 4, 20)
# 意义：把相邻两个嵌入向量拼在一起
x = emb.view(-1, 4, 2 * n_embd)   # (batch, 4, 20)
x = torch.tanh(x @ W1 + b1)        # (batch, 4, 200) — 每对字符组合成一个200维向量

# --- 第2层：再两两组合 ---
x = x.view(-1, 2, 2 * n_hidden)    # (batch, 2, 400)
x = torch.tanh(x @ W2 + b2)        # (batch, 2, 200)

# --- 第3层：最后组合成1个 ---
x = x.view(-1, 1, 2 * n_hidden)    # (batch, 1, 400)
x = torch.tanh(x @ W3 + b3)        # (batch, 1, 200)

# --- 输出层 ---
x = x.view(-1, n_hidden)            # (batch, 200)
logits = x @ W4 + b4                # (batch, 27)
```

**后置总结**：

- 每次 `view(-1, n_pairs, 2*dim)` 就是在手动提取卷积窗口——把相邻两个时间步的向量拼在一起。
- 然后 `@ W` 就是用共享权重做线性变换。
- 这两步合在一起**就是一维卷积**。
- 3层之后，每个输出向量都能"看到"所有8个输入字符。
- 代码量虽多，但每一步的几何意义非常清晰：从 $8 \rightarrow 4 \rightarrow 2 \rightarrow 1$ 的二叉树结构。

**变量名追踪**：

- `x` 在不同层代表不同东西：第1层是"字符对的组合"，第2层是"字符对的对的组合"，第3层是"最终汇总"。虽然名字一样，但语义逐层抽象。
- `emb` 和 `C[X]`：`emb` 是 `C[X]` 的结果，是嵌入查询的输出。后续 PyTorch 版本中等价于 `nn.Embedding` 层。

**踩坑记录**：

- 权重初始化：初版用 `torch.randn(...)` 没做缩放，导致初始 loss 极高（logits 值范围极大）。解决：除以 $\text{fan\_in}^{0.5}$ 做 Xavier 初始化。
- view 的维度计算：如果把 `(batch, 8, 10)` reshape 成 `(batch, 3, ...)` 会报错，因为8不能被3整除。必须确保分组数是整除的。

### 代码段3：等价的 PyTorch Conv1d 实现

**前置说明**：用手动代码理解了原理后，用 PyTorch 的 Conv1d 重写同样的操作。维度排列是关键——Conv1d 期望 $\text{(batch, channels, length)}$，而嵌入输出是 $\text{(batch, length, channels)}$。

```python
# 等价关系：
# 手动:  emb.view(-1, 4, 20) @ W1 + b1
# Conv1d: Conv1d(in_channels=10, out_channels=200, kernel_size=2, stride=2)

conv1 = nn.Conv1d(
    in_channels=n_embd,      # 输入通道数 = 嵌入维度(10)
    out_channels=n_hidden,   # 输出通道数 = 隐藏层大小(200)
    kernel_size=2,           # 窗口大小2（看相邻2个位置）
    stride=2,                # 步幅2（每次跳2个位置，输出长度减半）
)

# 前向传播
emb = C[Xtr]                        # (batch, 8, 10)
emb_permuted = emb.permute(0, 2, 1)  # (batch, 10, 8) — 关键！
out = conv1(emb_permuted)            # (batch, 200, 4)
```

**后置总结**：

- `permute(0, 2, 1)` 是最容易忘的地方——把"序列优先"变成"通道优先"。
- Conv1d 的 weight shape 是 $\text{(out\_channels, in\_channels, kernel\_size)} = (200, 10, 2)$。
- 和手动实现的 `W1.shape = (20, 200)` 本质相同，只是排列方式不同。
- **核心洞察**：卷积不是什么高深的数学概念，它就是"在每个位置用同一组权重做线性变换"。

### 代码段4：完整CNN模型（nn.Module封装）

**前置说明**：把上面的逻辑封装成一个标准的 PyTorch 模型，加入 BatchNorm。3层卷积 + 1个全连接头。

```python
class CNNLanguageModel(nn.Module):
    def __init__(self, vocab_size, n_embd, n_hidden):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, n_embd)
        
        # 3层卷积：kernel_size=2, stride=2
        # 维度变化：8 → 4 → 2 → 1
        self.conv1 = nn.Conv1d(n_embd, n_hidden, kernel_size=2, stride=2)
        self.bn1 = nn.BatchNorm1d(n_hidden)
        
        self.conv2 = nn.Conv1d(n_hidden, n_hidden, kernel_size=2, stride=2)
        self.bn2 = nn.BatchNorm1d(n_hidden)
        
        self.conv3 = nn.Conv1d(n_hidden, n_hidden, kernel_size=2, stride=2)
        self.bn3 = nn.BatchNorm1d(n_hidden)
        
        # 输出头
        self.fc = nn.Linear(n_hidden, vocab_size)
    
    def forward(self, x):
        # x: (batch, 8) 字符索引
        x = self.embedding(x)                 # (batch, 8, n_embd)
        x = x.permute(0, 2, 1)                # (batch, n_embd, 8)
        
        x = F.tanh(self.bn1(self.conv1(x)))   # (batch, n_hidden, 4)
        x = F.tanh(self.bn2(self.conv2(x)))   # (batch, n_hidden, 2)
        x = F.tanh(self.bn3(self.conv3(x)))   # (batch, n_hidden, 1)
        
        x = x.view(x.size(0), -1)             # (batch, n_hidden)
        logits = self.fc(x)                    # (batch, vocab_size)
        return logits

model = CNNLanguageModel(vocab_size, n_embd=10, n_hidden=64)
```

**后置总结**：

- 整个模型结构极其简洁：嵌入 → 3层卷积 → 全连接。
- 参数量远少于等效的 MLP（卷积层参数共享），但感受野覆盖了全部8个字符。
- 和 Day 5 手动实现的 MLP 对比：这里不需要把 $\text{(batch, 8, 10)}$ 拍成 $\text{(batch, 80)}$ 再做全连接。

**踩坑记录**：

- 最常犯的错误：忘记 `permute(0, 2, 1)`。Conv1d 期望通道维度在第2维（index 1），如果给 $\text{(batch, 8, 10)}$ 会让 Conv1d 认为序列长度为10、通道数为8，结果完全错误。
- 如果把 `F.tanh` 换成 `F.relu`，注意 dead neuron 问题（Day 3 讲过），BatchNorm 可以部分缓解。

### 代码段5：训练与采样

**前置说明**：标准训练循环。和之前的 MLP 训练结构一致，替换模型即可。

```python
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
batch_size = 32

for i in range(200000):
    # 取 mini-batch
    ix = torch.randint(0, Xtr.shape[0], (batch_size,))
    Xb, Yb = Xtr[ix], Ytr[ix]
    
    # 前向
    logits = model(Xb)
    loss = F.cross_entropy(logits, Yb)
    
    # 反向
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    
    if i % 10000 == 0:
        print(f'step {i}: loss {loss.item():.4f}')

# 验证集评估
model.eval()
with torch.no_grad():
    logits = model(Xdev)
    loss = F.cross_entropy(logits, Ydev)
    print(f'dev loss: {loss.item():.4f}')
```

**后置总结**：

- 验证集 loss 应该能达到约 2.0 左右，和之前的 MLP 模型相当或略好。
- 关键是用了更少的参数（卷积层参数共享）和更长的上下文（8 vs 3）。
- 训练速度可能比 MLP 快（计算图更规整，GPU 上卷积运算高效）。

**采样函数**（生成新名字）：

```python
def generate(model, max_len=20):
    model.eval()
    context = [0] * block_size  # 初始上下文全是'.'
    out = []
    for _ in range(max_len):
        x = torch.tensor([context])
        logits = model(x)
        probs = F.softmax(logits, dim=1)
        ix = torch.multinomial(probs, num_samples=1).item()
        if ix == 0:  # 遇到'.'就结束
            break
        out.append(itos[ix])
        context = context[1:] + [ix]  # 滑动窗口
    return ''.join(out)

print(generate(model))
```

## 模块4：数学公式与代码的对应

### 1. Conv1d 的核心公式

**数学公式**：

对于输入 $\mathbf{x} \in \mathbb{R}^{C_{\text{in}} \times L}$（$C_{\text{in}}$ 个通道，长度 $L$），权重 $\mathbf{w} \in \mathbb{R}^{C_{\text{out}} \times C_{\text{in}} \times K}$（$K$ 为 kernel\_size）：

$$
y[j, l] = \sum_{c=1}^{C_{\text{in}}} \sum_{k=0}^{K-1} w[j, c, k] \cdot x[c, \; l \cdot s + k] + b[j]
$$

**对应代码**：

```python
# PyTorch的一行操作：
out = conv(x)  # x: (batch, C_in, L) → (batch, C_out, L')
```

**对应变量**：

| 数学符号 | 代码中的变量 | 含义 |
|---|---|---|
| $x[c, \; l \cdot s + k]$ | 输入张量 | 通道 $c$、位置 $l \cdot s + k$ 处的值 |
| $w[j, c, k]$ | `conv.weight` | 输出通道 $j$、输入通道 $c$、窗口内偏移 $k$ 处的权重 |
| $b[j]$ | `conv.bias` | 输出通道 $j$ 的偏置 |
| $s$ | `stride` | 步幅 |
| $L'$ | 输出序列长度 | $L' = \lfloor (L - K) / s \rfloor + 1$ |

**直觉解释**：在每个输出位置，把窗口内的输入值按权重加权求和——这和一个小型全连接层做的事情一样，只不过这个小全连接层在序列上"滑动复用"。

---

### 2. 手动 reshape 与 Conv1d 的等价关系

**数学公式**（手动实现）：

$$
y[i] = \tanh\!\left( \left[\mathbf{x}_{2i} \;;\; \mathbf{x}_{2i+1}\right] \cdot \mathbf{W} + \mathbf{b} \right)
$$

其中 $[\;;\;]$ 表示向量拼接。

**对应代码对比**：

```python
# 手动实现：
x = emb.view(-1, 4, 2 * n_embd)   # 拼接相邻两个嵌入 [x_{2i} ; x_{2i+1}]
x = torch.tanh(x @ W1 + b1)        # 线性变换 x·W + b，再 tanh

# 等价的 Conv1d：
x = emb.permute(0, 2, 1)            # 通道维度前置
x = torch.tanh(conv1(x))            # Conv1d 自动完成窗口提取 + 线性变换
```

**直觉解释**：`view(-1, 4, 20)` 就是手动把卷积窗口内的元素提取出来拼在一起（即 $[\mathbf{x}_{2i} \;;\; \mathbf{x}_{2i+1}]$）；`@ W1` 就是手动做了卷积核的线性变换。Conv1d 只是把这些步骤封装进了一个高效的操作里。

---

### 3. 感受野公式

**数学公式**：对于 $L$ 层卷积，每层 $\text{kernel\_size}=k, \text{stride}=s$：

$$
\text{感受野} = 1 + L \times (k - 1) \times \prod_{i=0}^{L-1} s_i
$$

简化情况（$k=2, s=2$）：

$$
\text{感受野} = 2^{L}
$$

**直觉解释**：每多一层，"看到"的范围翻倍。3层看8个，4层看16个——指数增长。

---

### 4. BatchNorm 的数学公式

**前向传播**（训练时，对每个 channel $c$）：

$$
\mu_c = \frac{1}{B \cdot L} \sum_{b=1}^{B} \sum_{l=1}^{L} x_{b,c,l}
$$

$$
\sigma_c^2 = \frac{1}{B \cdot L} \sum_{b=1}^{B} \sum_{l=1}^{L} (x_{b,c,l} - \mu_c)^2
$$

$$
\hat{x}_{b,c,l} = \frac{x_{b,c,l} - \mu_c}{\sqrt{\sigma_c^2 + \epsilon}}
$$

$$
y_{b,c,l} = \gamma_c \cdot \hat{x}_{b,c,l} + \beta_c
$$

**对应代码**：

```python
x = F.tanh(self.bn1(self.conv1(x)))
# 等价于：
# 1) 计算每个 channel 的均值 mu 和方差 var
# 2) 标准化: (x - mu) / sqrt(var + eps)
# 3) 仿射变换: gamma * x_hat + beta
# 4) tanh 激活
```

**对应变量**：

| 数学符号                | 代码中的变量                                | 含义                        |
| ------------------- | ------------------------------------- | ------------------------- |
| $B$                 | `batch_size`                          | 批大小                       |
| $L$                 | 张量的第3维（序列长度）                          | 每个 channel 的时间步数          |
| $\mu_c, \sigma_c^2$ | `bn1.running_mean`, `bn1.running_var` | 每个 channel 的均值和方差（运行时统计量） |
| $\gamma_c, \beta_c$ | `bn1.weight`, `bn1.bias`              | 可学习的缩放和偏移参数               |
| $\epsilon$          | `bn1.eps`（默认 $10^{-5}$）               | 防止除零的小常数                  |

**直觉解释**：对每个 channel 独立地做"标准化 + 仿射变换"，相当于每门考试（channel）先按各自的标准把成绩归一化，再用可学习的参数重新缩放和偏移。这让每层的输入分布保持稳定，训练更平稳。

## 模块5：今日总结

### 收获

1. **卷积 = reshape + 共享权重的线性变换**。理解了这一点，卷积就不再是抽象概念，而是一个清晰的计算步骤。
2. **层次化组合比"拍扁"更聪明**。先组合局部再组合全局，用 $O(\log n)$ 深度覆盖 $O(n)$ 上下文，参数效率更高。
3. **感受野的指数增长**。3层 $\text{kernel\_size}=2$ 的卷积覆盖 $2^3 = 8$ 个输入位置，这和 Day 2 中 MLP 需要全连接才能覆盖形成鲜明对比。
