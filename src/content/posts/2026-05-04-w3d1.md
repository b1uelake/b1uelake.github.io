对于如何解决初始化的两个问题。第一次训练时的loss过高--由初始的W1和b1过高导致。tanh函数导致的大数被压缩至1/-1--调整W2和b2。

```python
W1 = torch.randn((n_embd * block_size, n_hidden), generator=g) * 0.2
b1 = torch.randn(n_hidden, generator=g) *0.01
W2 = torch.randn((n_hidden, vocab_size), generator=g) * 0.01
b2 = torch.randn(vocab_size, generator=g) * 0
```
那么现在的问题是，我们在式子后面加入的这些小数，到底多少最合适呢。
我们用一个例子来解决：

```python
x = torch.randn(1000, 10) 
w = torch.randn(10, 200)
y = x @ w
print(x.mean(), x.std())# 输出x的均值-0.0200，标准差0.9989
print(y.mean(), y.std())# 输出y的均值-0.0132，标准差3.2101

plt.figure(figsize=(20, 5))
plt.subplot(121)
plt.hist(x.view(-1).tolist(), 50, density=True);
plt.subplot(122)
plt.hist(y.view(-1).tolist(), 50, density=True);
```
y是x通过矩阵相乘后的结果，x的标准差为1，y的标准差是x的三倍，这会导致y经过tanh的压缩之后，出现很多1/-1
![w3d1-1778122032750.webp](/images/w3d1-1778122032750.webp)
y的标准差太大时，会出现很多1/-1，太小时，tanh就会不活跃，我们希望y符合高斯函数，及平均值在0附近，标准差稳定在 1.0 附近，从而保证深层网络依然能获得充足的信号。
所以我们可以通过调节w的值来让y的标准差≈1-->单位标准差。这个过程叫做<font color="#ffc000">归一化</font>。正确的做法是：w = torch.randn(10, 200) / 扇入值的根号-->w = torch.randn(10, 200) / `10**0.5`
 ![w3d1-1778122676495.webp](/images/w3d1-1778122676495.webp)
 ![w3d1-1778123255861.webp](/images/w3d1-1778123255861.webp)
 
 关于如何找到这些最理想的初始化值，有很多论文，这里推荐一篇。
 https://arxiv.org/abs/1502.01852

对于用各种压缩函数进行归一化，pytorch中有一个专门的函数来处理。
https://docs.pytorch.org/docs/2.11/nn.init.html#torch-nn-init

torch.nn.init.kaiming_normal_(_tensor_, _a=0_, _mode='fan_in'_, _nonlinearity='leaky_relu'_, _generator=None_)

使用 Kaiming 正态分布填充输入张量的值。
$$ \Large \text{std} = \frac{\text{gain}}{\sqrt{\text{fan\_mode}}} $$

gain的值根据不同的压缩函数选择。fan_mode包含fan_in和fan_out
**fan_in（扇入）**：指的是当前层每一个神经元接收到的**输入**连接数。
**fan_out（扇出）**：指的是当前层每一个神经元发送给下一层的**输出**连接数。

| nonlinearity      | gain                                                           |
| ----------------- | -------------------------------------------------------------- |
| Linear / Identity | $\displaystyle 1$                                              |
| Conv{1,2,3}D      | $\displaystyle 1$                                              |
| Sigmoid           | $\displaystyle 1$                                              |
| Tanh              | $\displaystyle \dfrac{5}{3}$                                   |
| ReLU              | $\displaystyle \sqrt{2}$                                       |
| Leaky Relu        | $\displaystyle \sqrt{\dfrac{2}{1 + \text{negative\_slope}^2}}$ |
| SELU              | $\displaystyle \dfrac{3}{4}$                                   |

那么此时我们直接改套用公式，得出代码

```python
g = torch.Generator().manual_seed(2147483647) # for reproducibility
C  = torch.randn((vocab_size, n_embd),            generator=g)
W1 = torch.randn((n_embd * block_size, n_hidden), generator=g) * (5/3)/((n_embd * block_size)**0.5) #* 0.2
b1 = torch.randn(n_hidden,                        generator=g) * 0.01
W2 = torch.randn((n_hidden, vocab_size),          generator=g) * 0.01
b2 = torch.randn(vocab_size,                      generator=g) * 0
```

但是在这么多年快速的AI发展后，我们对初始化的值的要求已经不再像之前那么高了。所以现在在真正的训练大模型中。

#### 对于归一化的问题，我们还有一种方法，就是<font color="#ffc000">正态分布的标准化</font>。

```python
# 前向传播
emb = C[Xb] # 将字符嵌入为向量
embcat = emb.view(emb.shape[0], -1) # 拼接向量
hpreact = embcat @ W1 + b1 # 隐藏层激活前计算
h = torch.tanh(hpreact) # 隐藏层
logits = h @ W2 + b2 # 输出层
loss = F.cross_entropy(logits, Yb) # 损失函数
```

我们先看看输入的的函数的图像，hpreact是我们设置的随机的正态分布。
![w3d1-1778139640375.webp](/images/w3d1-1778139640375.webp)
 标准化公式 (Z-score) 对于随机变量 $X$，其标准化后的值 $Z$ 为： $$ \large Z = \frac{X - \mu}{\sigma} $$
- $X$: 原始数据点
- $\mu$: 数据集的平均值（期望） 
- $\sigma$: 数据集的标准差 
- $Z$: 标准化后的得分，表示原始值距离平均值有多少个标准差

$\mu$ = hpreact.mean()
$\sigma$ = hpreat.std()

<u>所以更改后的代码为</u>

```python
# 前向传播
emb = C[Xb] # 将字符嵌入为向量
embcat = emb.view(emb.shape[0], -1) # 拼接向量
hpreact = embcat @ W1 + b1 # 隐藏层激活前计算
hpreact = (hpreact - hpreact.mean(0,keepdim=True)) / hpreact.std(0,keepdim=True)
h = torch.tanh(hpreact) # 隐藏层
logits = h @ W2 + b2 # 输出层
loss = F.cross_entropy(logits, Yb) # 损失函数
```

在前向传播的过程中，我们只希望初始化时的hpreact服从标准正态分布，但是在后续不断的迭代过程中，我们更希望他能有**区分能力**，根据学习到的内容改变。我们需要加入系数和偏置数让模型对标准正态进行后续改变。

更改后代码

```python
# MLP revisited
n_embd = 10 # 字符嵌入向量的维度
n_hidden = 200 # 多层感知器（MLP）隐藏层中的神经元数量。

g = torch.Generator().manual_seed(2147483647) # 加入固定种子，为了可重复性
C = torch.randn((vocab_size, n_embd), generator=g)
W1 = torch.randn((n_embd * block_size, n_hidden), generator=g) * (5/3)/((n_embd * block_size)**0.5)
b1 = torch.randn(n_hidden, generator=g) * 0.01
W2 = torch.randn((n_hidden, vocab_size), generator=g) * 0.01
b2 = torch.randn(vocab_size, generator=g) * 0

bngain = torch.ones((1, n_hidden)) # 1代表维度
bnbias = torch.zeros((1, n_hidden))

parameters = [C, W1, W2, b2,bngain,bnbias]
print(sum(p.nelement() for p in parameters)) # number of parameters in total
for p in parameters:
	p.requires_grad = True
```

```python
max_steps = 200000
batch_size = 32
lossi = []

for i in range(max_steps):
# 构建小批量数据
	ix = torch.randint(0, Xtr.shape[0], (batch_size,), generator=g)
	Xb, Yb = Xtr[ix], Ytr[ix] # 批量的输入X和标签Y
	
	# 前向传播
	emb = C[Xb] # 将字符嵌入为向量
	embcat = emb.view(emb.shape[0], -1) # 拼接向量
	hpreact = embcat @ W1 # + b1 b1在后续会被直接减去 # 隐藏层激活前计算
	bnmeani = hpreact.mean(0, keepdim=True)
	bnstdi = hpreact.std(0, keepdim=True)
	hpreact = bngain * (hpreact - bnmeani) / bnstdi + bnbias
	
	h = torch.tanh(hpreact) # 隐藏层
	logits = h @ W2 + b2 # 输出层
	loss = F.cross_entropy(logits, Yb) # 损失函数
	
	# 反向传播
	for p in parameters:
		p.grad = None # 清空梯度
	loss.backward() # 计算新梯度
	
	# 参数更新
	lr = 0.1 if i < 100000 else 0.01 # 阶梯式学习率衰减
	for p in parameters:
		p.data += -lr * p.grad # 现在 p.grad 一定不是 None
	
	# 记录统计数据
	if i % 10000 == 0: # 每隔一段时间打印日志
		print(f'{i:7d}/{max_steps:7d}: {loss.item():.4f}')
	lossi.append(loss.log10().item())
```

```python
# 训练结束后校准批量归一化层 ，可省去
torch with torch.no_grad(): 
# 将训练集数据前向传播  
emb = C[Xtr] 
embcat = emb.view(emb.shape[0], -1) 
hpreact = embcat @ W1  + b1 
# 在整个训练集上统计计算均值 / 标准差 
bnmean = hpreact.mean(0, keepdim=True) 
bnstd = hpreact.std(0, keepdim=True)
```

但是最后这一段代码有点麻烦，我们可以在训练时直接以平滑的方式处理，然后省去上面这一小段代码。解决方法和原因来自论文 https://arxiv.org/abs/1502.03167
简单来说，就是让新平均值和新标准差按照极高的比例参考之前的旧平均和旧标准差，新一轮的影响很小，从而让函数更平滑，并且省略了重新全部计算一次的繁琐步骤。

在定义MLP的代码中加入两行代码并增加到最后的参数列表中
```python
# running表示平滑，也就是在循环过程中自动进行平滑处理
bnmean_running = torch.zeros((1, n_hidden))
bnstd_running = torch.ones((1, n_hidden))

parameters = [C, W1, W2, b2, bngain, bnbias]
```

然后在前向传播中的hpreact = bngain * (hpreact - bnmeani) / bnstdi + bnbias后加入
代码
```python
with torch.no_grad(): # 不用计算返回的梯度
    bnmean_running = 0.999 * bnmean_running + 0.001 * bnmeani
    bnstd_running = 0.999 * bnstd_running + 0.001 * bnstdi
```
