## 深刻的理解反向传播

GitHub代码与练习网址： https://github.com/b1uelake/CMJ---zero-to-hero/blob/main/makemore/makemore-becoming%20a%20backprop%20ninja.ipynb


昨天我们完成了带有批量归一化（Batch Normalization）的多层感知机（MLP）网络搭建，并且通过交叉熵验证了其强大的拟合能力。今天的目标是“拆掉所有的辅助轮”——彻底弃用 PyTorch 提供的 `loss.backward()`，完全通过手动推导微积分并在张量（Tensor）级别编写全套的反向传播（Backpropagation）代码。


### 泄漏的抽象 (Leaky Abstraction)

- **一句话定义**：这是软件工程中的一个经典概念，指一个表面上将底层复杂细节完美封装的简单接口，但在某些边缘或异常情况下，底层物理现实或算法缺陷不可避免地“泄漏”出来，迫使使用者必须理解底层逻辑才能解决问题。

- **为什么要用它**：在深度学习中，PyTorch 的自动求导（Autograd）就是典型的“泄漏的抽象”。很多初学者认为只要搭建好网络，调用 `.backward()`，模型就会如魔法般自动收敛。但如果没有这个概念的武装，一旦你遇到梯度消失、梯度爆炸或网络不收敛的情况，你将毫无头绪。
- 如果不懂反向传播的底层流动机制，很容易写出这种看似合理实则完全背道而驰的代码。

#### 练习1

**反向传播中的核心逻辑：**

1. **链式法则（The Chain Rule）：** 任何复合过程的最终变化率，等于各环节局部变化率的连乘。
    
2. **梯度守恒（Conservation of Gradient）：** 如果一个变量通过两条路影响了最终结果，那么它的总梯度就是这两条路传回来的梯度之和（$\sum$）。
    
3. **维度一致性（Dimensional Consistency）：** 物理量必须在对应的维度上产生作用。如果前向传播时发生了“广播（Broadcasting）”，反向传播必发生“聚合（Sum）”。

### 第一战区：损失函数与 Softmax 的拆解

这是整个系统的“负压源”，所有梯度流动的动力都来自对“正确答案”的追求。

#### 1. 对数概率的导数 (`dlogprobs`)


```Python
dlogprobs = torch.zeros_like(logprobs)
dlogprobs[torch.arange(n), Yb] = -1.0 / n
```

- **第一性原理说明**：前向传播中，$Loss = -\frac{1}{N} \sum \log(p_i)$。
    
- **解析**：损失函数只关心正确类别的对数概率。对于正确类别，$\frac{\partial Loss}{\partial \log(p)} = -1/N$。对于其他 26 个错误类别，它们的变化对 Loss 的初始贡献是 0。**物理直觉**：这就像在 27 个按钮中，我们只对那个“正确按钮”施加了一个向下拉（减小 Loss）的拉力。


#### 2. 概率的倒数导数 (`dprobs`)



```Python
dprobs = (1.0 / probs) * dlogprobs
```

- **第一性原理说明**：已知 $y = \log(x)$，则 $\frac{dy}{dx} = \frac{1}{x}$。
    
- **解析**：为了让 $\log(p)$ 增加，底层概率 $p$ 必须增加。这里的 $1/p$ 是一个“放大器”：如果当前的概率 $p$ 已经很小，那么同样的 $d\log(p)$ 会要求 $p$ 做出更剧烈的改变。


#### 3. 求和逆操作 (`dcounts_sum_inv`)

```Python
dcounts_sum_inv = (counts * dprobs).sum(1, keepdim=True)
```

- **第一性原理说明**：前向是 $probs = counts \times counts\_sum\_inv$。这是 $f(x,y) = x \cdot y$ 的乘法求导。
    
- **解析**：由于 `counts_sum_inv` 是被广播（Broadcast）到每一行的 27 个元素上的，根据**能量守恒原理**，它在 27 个位置上产生的微小影响必须全部收集回来，因此使用 `.sum(1)`。


#### 4. 计数项的初步导数 (`dcounts`)

```Python
dcounts = counts_sum_inv * dprobs  # 路径 A
```

- **第一性原理说明**：同样源于乘法求导 $\frac{\partial(a \cdot b)}{\partial a} = b$。

#### 5. 倒数函数求导 (`dcounts_sum`)

```Python
dcounts_sum = (-counts_sum**-2) * dcounts_sum_inv
```

- **第一性原理说明**：已知 $y = x^{-1}$，则 $\frac{dy}{dx} = -x^{-2}$。这是微积分的基础幂规则。

#### 6. 聚合函数的逆流 (`dcounts`)

```Python
dcounts += torch.ones_like(counts) * dcounts_sum  # 路径 B
```

- **第一性原理说明**：前向是 $counts\_sum = \sum counts$。
    
- **解析**：这是一个“分叉”点。`counts` 的总梯度 = 路径 A（直接贡献给 probs）+ 路径 B（通过影响总和来影响 probs）。**物理直觉**：由于每个 count 的增加都会导致总和 `counts_sum` 增加，所以梯度像电流一样平摊给每一个参与求和的 count。

### 第二战区：激活函数与线性映射

这是信号的调制与空间变换区。

#### 7. 指数函数求导 (`dnorm_logits`)

```Python
dnorm_logits = counts * dcounts
```

- **第一性原理说明**：已知 $y = e^x$，则 $\frac{dy}{dx} = e^x$。
    
- **解析**：前向是 `counts = exp(norm_logits)`。因为 $e^x$ 的导数就是它本身，所以代码里直接写 `counts * dcounts`。这体现了指数函数的自增长特性。

#### 8. Tanh 门控求导 (`dhpreact`)

```Python
dhpreact = (1.0 - h**2) * dh
```

- **第一性原理说明**：已知 $y = \tanh(x)$，则 $\frac{dy}{dx} = 1 - y^2$。
    
- **解析**：Tanh 的物理特性是“饱和”。当输出 $h$ 接近 1 或 -1 时，梯度 $1-h^2$ 趋近于 0。这从第一性原理上解释了为什么深层网络会出现**梯度消失**：因为门关上了，压力传不过去。

### 第三战区：批量归一化（BatchNorm）的精细解构

单个样本如何影响群体。
#### 9. 归一化输出对方差的梯度 (`dbnvar`)

```Python
dbnvar = (-0.5 * (bnvar + 1e-5)**-1.5) * dbnvar_inv
```

- **第一性原理说明**：复合函数求导。前向是 $y = (x+\epsilon)^{-0.5}$。
    
- **解析**：这是标准差倒数的导数。负号表示方差越大，归一化后的信号强度反而越弱（被抑制）。

#### 10. 均值节点的梯度汇聚 (`dbnmeani`)


```Python
dbnmeani = (-dbndiff).sum(0)
```

- **第一性原理说明**：前向是 $diff = x - mean$。
    
- **解析**：`mean` 是从 32 个样本中计算出来的，因此它受到 32 个样本回传压力的总和。由于公式里是减去均值，所以系数是 -1。

#### 11. 输入节点的最终汇聚 (`dhprebn`)

```Python
dhprebn = dbndiff.clone()  # 路径 1：直接贡献
dhprebn += (1.0 / n) * torch.ones_like(hprebn) * dbnmeani # 路径 2：通过均值贡献
```

- **第一性原理说明**：变量 $x$ 既参与了差值计算，又参与了均值计算。
    
- **解析**：每个样本对均值的贡献是 $1/N$。因此，来自均值的压力 `dbnmeani` 需要平均分配回每一个样本。

### 第四战区：空间投影（矩阵乘法）

在这里，我们将梯度从“特征空间”转回“输入空间”。

#### 12. 权重矩阵梯度 (`dW1`)

```Python
dW1 = embcat.T @ dhprebn
```

- **第一性原理说明**：线性变换 $Y = XW$ 的偏导。
    
- **解析**：为什么是 `X.T @ dY`？从第一性原理看，权重 $W_{ij}$ 连接的是输入 $X_i$ 和输出 $Y_j$。因此它的变化率取决于它所连接的那个输入的强度（$X_i$）以及它所承受的输出端的压力（$dY_j$）。矩阵乘法自动完成了这种配对累加。

#### 13. 嵌入层梯度累加 (`dC`)

```Python
for k in range(Xb.shape[0]):
    for j in range(Xb.shape[1]):
        ix = Xb[k, j]
        dC[ix] += demb[k, j]
```

- **第一性原理说明**：索引操作 $y = C[index]$。
    
- **解析**：这是一个典型的“多对一”映射。词表中同一个词（如字母 'e'）在 32 个句子中可能出现了 10 次。这 10 个位置产生的梯度都在影响同一个物理实体——词表中 'e' 的向量。因此，必须将这 10 个压力全部加起来。

## 模块 4：数学公式与代码的对应（物理意义版）

|**计算步骤**|**物理意义**|**数学公式 (f′)**|**核心代码逻辑**|
|---|---|---|---|
|**Sum**|压力广播|$\frac{\partial \sum x}{\partial x} = 1$|`torch.ones_like(x) * dy`|
|**Broadcast**|压力聚合|$\frac{\partial (x+b)}{\partial b} = \sum 1$|`dy.sum(dim=...)`|
|**$x^2$**|线性缩放|$\frac{\partial x^2}{\partial x} = 2x$|`2 * x * dy`|
|**$exp$**|信号放大|$\frac{\partial e^x}{\partial x} = e^x$|`y * dy`|
|**$1/x$**|信号抑制|$\frac{\partial (1/x)}{\partial x} = -1/x^2$|`-y**2 * dy`|

## 模块 5：今日总结

- **收获**：
    
    1. **理解了梯度的“物质性”**：梯度不再是抽象的数字，而是像水流或压力一样，在计算图中流动。遇到 `sum` 就分散，遇到“分叉”就汇聚。
        
    2. **识破了框架的“魔法”**：PyTorch 的 `.backward()` 其实就是一套极其严密的、基于维度的“拼图系统”。只要理解了维度匹配和路径求和，没有任何层是推不出来的。
        
    3. **BatchNorm 的祛魅**：以前觉得 BN 很神秘，现在看来它就是一系列加减乘除的组合，它的梯度复杂仅仅是因为它在 Batch 维度上做了一次全员关联。
